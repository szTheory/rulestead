# Milestone v0.1.0: First Polished Hex Release

**Status:** ✅ SHIPPED 2026-04-24
**Close Mode:** Acknowledged debt close
**Phases:** 1-8
**Total Plans:** 50

## Overview

Rulestead v0.1.0 shipped the first complete sibling-package release shape: the `rulestead` runtime library, the `rulestead_admin` mounted LiveView admin surface, the supporting installer and host-app seams, and the documentation and release-engineering layer required to publish and verify the release.

This milestone was closed with two explicit deferred items that were addressed in Phase 13:

- **[CLOSED in Phase 13]** Phase 7 sibling-package simulation test seeding has been updated with an actor-aware helper, and the Phase 7 verification artifact has moved from `gaps_found` to `passed`.
- **[ACKNOWLEDGED in Phase 13]** The live post-publish verification wave remained blocked as both `rulestead` and `rulestead_admin` APIs continued to return `404` on Hex, officially logging the delay in package propagation.

## Accomplishments

1. Established the sibling-package monorepo, linked-version release flow, CI foundation, and contributor/docs scaffolding.
2. Shipped the Ecto authoring model, typed public error envelope, real and fake stores, and the first `mix rulestead.install` slice.
3. Locked the deterministic pure evaluator, explain surface, and property-tested bucketing/runtime contracts.
4. Added snapshot publication, ETS runtime caching, refresh orchestration, backup restore, and the public telemetry event catalog.
5. Delivered Phoenix/LiveView/Oban integration seams, a full installer, fake-backed test helpers, and golden-diff install verification.
6. Completed the mounted admin UI plus v0.1.0 docs, API stability inventory, verification trio, and gated publish/drift automation.

## Milestone Summary

**Decimal Phases:** None

**Key Decisions:**

- Keep the project as linked sibling packages from day one: `rulestead` for runtime, `rulestead_admin` for mounted operator UI.
- Treat payload-first evaluation, deterministic bucketing, explicit context, and snapshot-backed runtime reads as non-negotiable core contracts.
- Keep the public admin contract narrow at the package boundary; do not stabilize internal LiveView module names or DOM details.
- Require scripts-first verification and a gated publish flow with one explicit human approval before Hex credentials are used.

**Issues Resolved:**

- Closed the runtime hot-path, snapshot publication, telemetry catalog, installer, and host-app integration gaps needed for a first public release.
- Delivered the mounted admin UI across lifecycle, simulation, rollout, kill-switch, audit, and security/redaction workflows.
- Replaced placeholder docs with release-grade guides, recipes, cheatsheet, API stability inventory, and release verification tooling.

**Issues Deferred:**

- **[CLOSED in Phase 13]** Phase 7 sibling-package verification gap: the simulation test helper was updated to seed rulesets with actor metadata, closing the verification gap.
- **[ACKNOWLEDGED in Phase 13]** Live Hex publish evidence: both package APIs returned `404` during the final verification wave. The delay in propagation was captured as the expected blocked state.

**Technical Debt Incurred:**

- Milestone close accepted deferred verification debt in exchange for archiving the shipped planning state and moving milestone management forward.
- `REL-03` and `REL-04` are implemented in code and workflow form, but their live evidence capture still depends on post-close publication work.
