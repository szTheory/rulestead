# Roadmap: v0.5.0 - Advanced Delivery & Distributed Scale

## Phases

- [x] **Phase 19: Redis Storage & Caching Adapter** - Introduce an external, high-throughput caching layer.
- [x] **Phase 20: PubSub Distributed Invalidation** - Eliminate polling bottlenecks by streaming deltas and cache invalidation notices across nodes.
- [x] **Phase 21: Infrastructure Observability UI** - Surface distributed topology health and cache freshness in the operator Admin UI.

## Phase Details

### Phase 19: Redis Storage & Caching Adapter
**Goal**: The runtime engine can pull state from a centralized Redis instance, avoiding Ecto DB bottlenecks at scale.
**Depends on**: None
**Requirements**: STO-01, STO-02
**Plans**: 2 plans
- [x] 19-01-PLAN.md — Redis Store Adapter
- [x] 19-02-PLAN.md — Redis Publisher & Degraded Mode
**Success Criteria** (what must be TRUE):
  1. Operators can configure Rulestead to use Redis as the primary external state cache.
  2. The system handles Redis connection failures gracefully, falling back to local ETS or previous snapshot states.

### Phase 20: PubSub Distributed Invalidation
**Goal**: Nodes in a cluster receive instant push notifications when flag data changes, avoiding cross-node drift and expensive polling loops.
**Depends on**: Phase 19
**Requirements**: INV-01, INV-02
**Plans**: 3 plans
- [x] 20-01-PLAN.md — Notifier Seam & Publish Hooks
- [x] 20-02-PLAN.md — Runtime Invalidation Flow & Telemetry
- [x] 20-03-PLAN.md — Installer PubSub Scaffolding
**Success Criteria** (what must be TRUE):
  1. A mutation in the Admin UI triggers a PubSub broadcast.
  2. All connected nodes receive the broadcast and instantly invalidate/refresh their local ETS snapshot cache.
  3. The system supports `Phoenix.PubSub` as a first-class citizen.

### Phase 21: Infrastructure Observability UI
**Goal**: SREs and Operators can visually confirm that cache states across the cluster are healthy and synchronized.
**Depends on**: Phase 20
**Requirements**: INF-01, INF-02
**Plans**: 2 plans
- [x] 21-01-PLAN.md — Runtime Health Projection & Telemetry Compatibility
- [x] 21-02-PLAN.md — Admin Diagnostics UI
**Success Criteria** (what must be TRUE):
  1. The Admin UI includes an "Infrastructure Health" or "System Status" diagnostic panel.
  2. Cache age and connection statuses (Ecto, Redis, PubSub) are reported and visible.
  3. Clear telemetry events are emitted for cache invalidations and syncs.
