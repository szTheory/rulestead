# Requirements: v0.5.0 - Advanced Delivery & Distributed Scale

## Advanced Storage (STO)
- **STO-01**: Build a Redis storage adapter that implements the core storage behaviour, allowing Rulestead to optionally bypass Ecto/Postgres for high-throughput reads.
- **STO-02**: Ensure the runtime evaluator seamlessly falls back to local ETS caches and maintains graceful degradation if the external Redis cache becomes unavailable.

## Distributed Invalidation (INV)
- **INV-01**: Implement PubSub-based streaming deltas (push invalidation) to instantly notify all connected Phoenix nodes of flag or rule mutations.
- **INV-02**: Decouple the PubSub invalidation logic from strict Phoenix dependencies where possible, enabling usage in pure Elixir/Plug architectures while supporting native `Phoenix.PubSub` integration out of the box.

## Infrastructure Observability (INF)
- **INF-01**: Expose cache age, sync latency, and adapter connection health in the Admin UI so operators have explicit visibility into distributed state drift.
- **INF-02**: Emit new Telemetry events (`[:rulestead, :sync, :delta_received]`, `[:rulestead, :cache, :invalidation]`) to hook into existing host app metrics.
