# Plan 59-03: Audience Confirm Governance Actions

<objective>
Implement confirm-step enforcement: direct apply vs submit change request vs blocked, with approval expectations and ADM-03 visibility gates.
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-59-04: Apply bypass on governed confirm | high | No `phx-submit="apply"` when `governance_mode == :change_request` |
| T-59-05: Submit CR with hidden refs | high | `:blocked` when `visibility_tier` partial/denied — no submit form |
| T-59-06: Stale preview submit | high | Re-run `load_governance_context` on confirm mount; fingerprint match in command |
</threat_model>

In `edit_confirm.ex`:

**Mount/load:** After preview loaded, call `Governance.load_governance_context(socket, preview, operation: :update)` and `Governance.approval_expectation_assigns(socket, audience_key)`.

**Render branches:**

| `@governance_mode` | UI |
|--------------------|-----|
| `:unrestricted` / `:direct_apply` | Existing `phx-submit="apply"` form (Apply update) |
| `:change_request` | `blast_radius_panel` compact; NO apply form; `phx-submit="submit_change_request"` button **Submit change request**; show required approvals + self-approval line |
| `:blocked` | Panel with indeterminate copy; NO apply; NO submit; remediation + Back to preview link |

When `can_submit?` false: render `OperatorComponents.capability_explanation` title="Change request required".

**`handle_event("submit_change_request", ...)`:**

```elixir
with :ok <- validate_reason(reason),
     {:ok, preview} <- ensure_preview(socket),
     {:ok, audience} <- fetch_audience(...),
     :ok <- ensure_governance_mode(socket, :change_request),
     approval_requirement <- build_approval_requirement(socket, :apply_audience_mutation, audience_key),
     command_map <- audience_mutation_command_map(socket, preview, audience, :update),
     {:ok, %{change_request: cr}} <- Rulestead.submit_change_request(
       Command.SubmitChangeRequest.new(%{
         action: :apply_audience_mutation,
         environment_key: socket.assigns.current_environment.key,
         resource_type: "audience",
         resource_key: audience_key,
         command: command_map,
         approval_requirement: approval_requirement
       }, actor: socket.assigns.current_actor, reason: reason)
     ) do
  {:noreply,
   socket
   |> put_flash(:info, "Change request submitted. Audience definition is unchanged until this request is approved and executed.")
   |> push_navigate(to: Session.current_path(socket, "#{mount}/change-requests/#{cr.id}"))}
end
```

Extract shared helpers to `governance.ex` if needed: `build_approval_requirement/4`, `audience_mutation_command_map/4`.

**Indeterminate:** Use verdict label **Cannot evaluate safely**; show top `breach_reason` remediation; link to preview with `Session.current_path`.

Create `edit_confirm_governance_test.exs`:
- Custom router policy module `RulesteadAdmin.GovernanceTestPolicy` with `change_request_required?` → true, `allow_self_approval?` configurable
- Prod + 3 refs: refute `Apply update`, assert `Submit change request`
- Submit form → assert redirect path contains `/change-requests/` and flash text `unchanged until`
- Partial visibility: stub/list deps with `hidden_reference_count > 0` (Fake control or policy denying some flags) → refute Submit, assert `Cannot evaluate safely` or hidden refs copy
