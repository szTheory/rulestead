<objective>
Wire `BlastRadiusThreshold.validate_protected_apply/3` into Fake `do_apply_audience_mutation/2`, replace the placeholder `ensure_protected_audience_confirmation/1` gate, and extend Fake contract tests for threshold behavior per D-02 and D-06.
</objective>

</threat_model>

  <action>
In `rulestead/lib/rulestead/fake.ex`:

1. Add alias: `alias Rulestead.Governance.BlastRadiusThreshold`

2. In `do_apply_audience_mutation/2`, replace the `with` chain step:
   - **Remove:** `:ok <- ensure_protected_audience_confirmation(command)`
   - **Add after** `:ok <- ensure_fresh_audience_preview(command, current_preview)`:
     ```elixir
     :ok <- validate_blast_radius_threshold(command, current_preview)
     ```

3. Implement `validate_blast_radius_threshold/2`:
   ```elixir
   defp validate_blast_radius_threshold(command, current_preview) do
     dependency_entries = audience_dependency_entries(current_preview, command)

     BlastRadiusThreshold.validate_protected_apply(
       command,
       current_preview,
       dependency_entries: dependency_entries
     )
   end
   ```

4. Extend `BlastRadiusThreshold.validate_protected_apply/3` opts if needed to accept `dependency_entries:` keyword.

5. On threshold block (when validate returns error), before returning to caller:
   - Append audit event with metadata including:
     - `"blast_radius_verdict"` (from error metadata)
     - `"blast_radius_breach_reasons"` (from error metadata)
     - `"preview_fingerprint"`, `"reference_count"`
   - Use event type pattern similar to dependency blocked path (e.g. existing `blocked_audience_event_type/1` or new `"audience_mutation.blast_radius_blocked"`)

6. **Delete or deprecate** `ensure_protected_audience_confirmation/1` — remove both function heads. If `protected_shared_targeting?` is still on command struct, pass it as metadata amplifier to assess opts (optional `shared_targeting?: true` in assess input) but NOT as sole gate trigger.

7. Keep dependency validation flow unchanged AFTER threshold passes.
