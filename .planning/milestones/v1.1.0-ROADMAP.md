# Milestone v1.1.0: Tenancy Helpers & Validation

**Status:** ✅ SHIPPED 2026-05-23
**Phases:** 29-34
**Total Plans:** 12

## Overview

`v1.1.0` closes the first deliberate post-GA milestone for Rulestead. This release adds the smallest coherent tenancy-aware seam that fits the shipped linked-version, two-package product shape: explicit tenant scope now survives runtime helpers, mounted admin flows, compare and promotion review paths, saved plans, replay/apply boundaries, and audit provenance without widening into tenant-partitioned storage, environment-per-tenant topology, or standalone admin drift.

## Milestone Summary

**Key Decisions:**

- Activated tenancy as an explicit `v1.1.0` post-GA milestone instead of silently inheriting old pre-GA carryover.
- Kept tenancy inside bounded helper seams, reviewed-artifact validation, and explicit metadata contracts rather than expanding into tenant-partitioned topology.
- Preserved the linked-version sibling-package release model and avoided standalone `rulestead_admin` drift while extending tenancy through mounted admin and public replay surfaces.

**Issues Resolved:**

- Explicit tenant scope now survives mounted admin session, compare, promotion-plan, replay, and apply flows.
- Audit and scheduled execution paths now preserve bounded tenant provenance automatically instead of depending on caller-authored metadata.
- The compare summary-to-drill-in path now preserves preview identity through `compare_token`.
- Milestone traceability is consistent again after restoring the missing Phase 30 summary and verification artifacts.

**Issues Deferred:**

- Lifecycle and ownership work remain the strongest follow-on milestone candidate after the bounded tenancy seam shipped cleanly.

**Technical Debt Incurred:**

- Phases 29 and 30 still use older validation-file metadata conventions; the milestone audit records this as non-blocking documentation hygiene rather than a release blocker.
