# Requirements: Rulestead — v1.16 Brand-Faithful UI Iteration

**Defined:** 2026-06-12
**Core Value:** Phoenix teams can safely gate, roll out, and explain runtime decisions — booleans, variants, and remote config — with 15-minute quickstart, deterministic evaluation, and a calm admin UI that operators, support, and SRE can all trust at 3am.

**Milestone goal:** Make every rendered Rulestead-owned surface faithfully reflect the v1.15 identity in light, dark, and system modes, while keeping FleetDesk clearly host-branded. This is a bounded post-GA UI/brand quality milestone: no runtime APIs, schema changes, package-version changes, release workflow changes, component framework adoption, or `rulestead_admin` publish preparation.

## v1 Requirements

### Brand / UI Conformance (BUI)

- [x] **BUI-01**: A repo-local UI/brand audit defines the conformance matrix across admin route clusters, demo launcher, FleetDesk host app, brandbook, and static fixtures. It records the host-vs-Rulestead brand boundary and prohibits palette/logo re-litigation.
- [x] **BUI-02**: Static fixtures and guardrails reflect the shipped v1.15 identity: design/theme harnesses expose the winner wordmark, stale token literals are removed, copied logo assets are drift-checked against `brandbook/assets/logo/`, and contrast/token/brandbook guards run in the normal lint path.
- [x] **BUI-03**: Shared admin primitives respect the frozen mineral brand system in both light and dark modes, including primary-button foreground contrast, Stead Blue-derived focus/selection rings, status semantics that are not color-only, and no host-app token leakage.
- [x] **BUI-04**: Admin workflow clusters render with the v1.15 shell identity and remain operator-dense, keyboard-oriented, responsive, and scannable across Build & release, Explain & diagnose, Review & approve, and destructive per-flag flows.
- [x] **BUI-05**: Demo surfaces are aligned without confusing brands: the Phoenix demo launcher uses Rulestead logo/theme chrome, while FleetDesk remains visibly host-branded with its own light/system-dark visual system and no stale teal/Phoenix boilerplate.
- [x] **BUI-06**: Browser evidence proves the milestone: curated Playwright screenshots/assertions cover admin route clusters, demo launcher, FleetDesk, fixtures, light/dark/system modes, desktop/mobile widths, logo visibility, theme controls, and horizontal-overflow absence.

## Out of Scope

| Feature | Reason |
|---------|--------|
| New runtime APIs, schemas, migrations, or storage contracts | This is a UI/brand fidelity milestone only. |
| Palette redesign or logo redraw | v1.14/v1.15 froze palette and identity; this milestone may only correct evidence-proven semantic token drift using existing palette colors. |
| Making FleetDesk look like Rulestead | FleetDesk is the host/example product. The demo must teach the mounted-library boundary, not blur it. |
| New component framework or CSS build pipeline | Existing hand-authored scoped CSS and current Phoenix/Next asset pipelines are sufficient. |
| Pixel baseline snapshots for every route | Broad evidence artifacts are required; brittle all-screen baselines are not. |
| `rulestead_admin` publishing prep | The admin stub remains unpublished until the release topology decision says otherwise. |
| Phase 8-only or future v2 docs | Existing AGENTS.md constraint remains active. |
