    Create `scripts/check_contrast.py` mirroring the single-file, stdlib-only, exits-0-or-1 style of `check_synced_pair.py`. The file must have:

    1. Shebang `#!/usr/bin/env python3` and a module docstring explaining its purpose (WCAG 2.x contrast + OKLCH hue-angle verification for Phase 95 palette reconciliation; see `check_synced_pair.py` for format precedent).

    2. Three pure functions using only `math` and `sys` (both stdlib):
       - `linearize(c_byte)` — sRGB channel byte to linear light per WCAG 2.1. Formula: first normalize `c = c_byte / 255`; then if `c ≤ 0.04045` use `c / 12.92`, else `((c + 0.055) / 1.055) ** 2.4`. (Normalize by 255 exactly once — do NOT divide by 255 again inside the gamma branch.)
       - `relative_luminance(r, g, b)` — WCAG 2.x luminance: `0.2126 * linearize(r) + 0.7152 * linearize(g) + 0.0722 * linearize(b)`.
       - `contrast_ratio(hex1, hex2)` — parse both hex strings, compute both luminances, return `(lighter + 0.05) / (darker + 0.05)`.

    3. One OKLCH function:
       - `rgb_to_oklch(r, g, b)` — full Ottosson M1+M2 chain: sRGB bytes → linear → XYZ D65 (M1 coefficients from RESEARCH.md) → LMS → cube-root LMS → OKLab → OKLCH. Return `(L, C, H_degrees)` where H is `math.degrees(math.atan2(b_ok, a)) % 360`.

    4. A `hue_drift(hex_before, hex_after)` helper returning `abs(H_after - H_before)` with 360-wraparound correction (if diff > 180, use 360 - diff).

    5. A `SELF_TEST` section that runs four known-good anchor assertions and calls `sys.exit(1)` with an error message if any fails:
