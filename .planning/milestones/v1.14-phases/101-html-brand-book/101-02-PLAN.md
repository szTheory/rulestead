<threat_model>
ASVS level 1 posture is sufficient because the output is static HTML with local inline
assets and read-only controls.

Threats and mitigations:
- Source-truth drift: section prose or token values could be duplicated in generator code. Mitigate by rendering overview, voice, color, typography, logo, layout, iconography, imagery, motion, and UI-writing surfaces from extracted source sections, `VOICE.md`, `COPY.md`, `tokens.json`, `tokens.css`, and asset manifests.
- Unsafe HTML/SVG embedding: richer preview rendering could reintroduce unsafe SVG or duplicate ID problems. Mitigate by using the Plan 101-01 SVG sanitizer/prefixer for every preview and by adding visible source refs instead of external preview paths.
- External asset dependency: Google Fonts links are allowed, but required previews and core page behavior must not depend on external network access. Mitigate with system fallbacks, inline CSS/JS, no `script src`, and inline committed SVG previews.
- Broken no-JS/browser rendering: theme switching and any preview affordance could hide content without JavaScript. Mitigate by rendering all nav, source refs, sections, and previews in baseline HTML; JS only updates `data-theme`, `aria-checked`, and optional copy affordances.
- Contrast and focus regression: generated CSS could ignore Phase 95 AA palette. Mitigate with wrapper-scoped light and dark variables from `tokens.json`, visible `:focus-visible` rules, and no Signal Gold normal text.
- Premature milestone closeout: this plan does not update milestone status files.
- High severity block condition: any generated output that requires JavaScript to read content, declares color tokens on `:root`, `html`, or `body`, or imports external JavaScript blocks this plan.
</threat_model>
