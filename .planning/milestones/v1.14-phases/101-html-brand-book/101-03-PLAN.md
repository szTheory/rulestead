<threat_model>
ASVS level 1 posture is sufficient because this plan adds local guard scripts and docs.

Threats and mitigations:
- Source-truth drift: generated HTML could be edited by hand. Mitigate with `scripts/check_brandbook_html.py`, importing `render_brandbook()` and byte-comparing the rendered output against `brandbook/index.html`.
- Unsafe HTML/SVG embedding: checker might only compare drift and miss unsafe content. Mitigate with static assertions for forbidden patterns, duplicate inline SVG IDs, required source refs, and local-link validity.
- External asset dependency: generated page could add external scripts or required `img src` previews later. Mitigate with checker rejections for `<script src=` and `<img src=` in required preview paths.
- Broken no-JS/browser rendering: checker cannot prove visual quality, but it can enforce baseline static structure: wrapper, landmarks, required section IDs, inline SVGs, and source refs. Browser evidence is planned in 101-04.
- Budget bypass: inline SVG previews could grow the page silently. Mitigate with `HTML_BUDGET_BYTES = 262144` in the checker and a matching `brandbook/BUDGET.md` row.
- Premature milestone closeout: this plan adds enforcement only and does not mark BOOK requirements or v1.14 shipped.
- High severity block condition: any checker that rewrites source files, skips drift on render failure, relaxes existing SVG budgets, or requires Node/Playwright in `scripts/ci/lint.sh` blocks this plan.
</threat_model>

    Add `scripts/check_brandbook_html.py` as an executable Python 3 stdlib guard.

    Concrete script contract:
    - Define `REPO_ROOT = Path(__file__).resolve().parents[1]`.
    - Define `OUTPUT = REPO_ROOT / "brandbook" / "index.html"`.
    - Define `HTML_BUDGET_BYTES = 262144`.
    - Import `render_brandbook` from `gen_brandbook_html`.
    - Define `REQUIRED_SECTION_IDS` with exactly: `overview`, `voice-and-messaging`, `color`, `typography`, `logo`, `layout-and-components`, `iconography-and-imagery`, `motion`, `assets-and-maintenance`.
    - Define final logo source refs and specimen source refs matching the 101-01 manifests.
    - If `brandbook/index.html` is missing, print `ERROR: brandbook/index.html is missing; run python3 scripts/gen_brandbook_html.py` and return 1.
    - Render expected HTML in memory with `render_brandbook(REPO_ROOT)`.
    - Byte/text-compare actual vs expected. On mismatch, print `BRANDBOOK HTML DRIFT DETECTED`, then a concise `difflib.unified_diff` with fromfile `brandbook/index.html` and tofile `generated`.
    - Enforce size by measuring `len(actual.encode("utf-8"))`; if over budget, print `BRANDBOOK HTML BUDGET EXCEEDED: brandbook/index.html is N bytes (limit: 262144)` and return 1.
    - Add static assertions that fail with `ERROR:` messages for: missing required section IDs; section IDs out of order; missing visible final logo/specimen filenames; `<script src=`; `<img src=`; `base64`; `<image`; `<foreignObject>`; duplicate inline SVG IDs; local non-fragment `href` links that do not resolve from `brandbook/`; and missing trailing newline.
    - On success, print exactly `BRANDBOOK HTML SYNCED (N bytes)` and return 0.
  </action>
