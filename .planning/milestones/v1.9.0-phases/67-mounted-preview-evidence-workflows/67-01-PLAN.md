In `rulestead_admin/lib/rulestead_admin/components/audience_components.ex`:

1. Add module attribute `@sample_display_limit 10`.

2. Add private helpers:
   - `fetch_preview(preview, key)` — `Map.get(preview, key) || Map.get(preview, to_string(key))`
   - `sample_evidence_list/1` — `List.wrap(fetch_preview(preview, :sample_evidence))`
   - `impression_evidence_map/1` — normalize to map or `%{}`
   - `display_samples/1` — take first 10, return `{visible, remaining_count}`
   - `sample_row/1` — extract allowlisted fields with atom/string key tolerance
   - `format_matched?/1` — "Yes" / "No" for boolean

3. Inside `impact_preview/1` template, **after** affected references and **before** closing `</section>`:

   **Sample cohort** (`:if={samples != []}` where `samples = sample_evidence_list(@preview)`):
   - `<h3>Sample cohort</h3>`
   - Table columns: Actor, Targeting key, Matched?, Reason
   - `:for={row <- display_samples(samples).visible}`
   - `<p :if={remaining > 0}>+<%= remaining %> more</p>`

   **Impression summary** (`:if={impression != %{}}`):
   - `<h3>Impression summary</h3>`
   - Show `window_label`, `sampled_impressions`, `matched_impressions` when present
   - Optional `variant_breakdown`: inline `<ul>` with variant + count per entry (support atom or string keys)

4. Omit both sections when lists/maps empty — do not render empty headings.
