# Plan 65-02: ImpactPreview Schema v2 And Evidence Fingerprints

<objective>
Bump `ImpactPreview` to schema v2 with `impression_evidence`, extended fingerprint token, and basis/uncertainty messages per CONTEXT D-04, D-05, IMP-06.
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-65-03: Evidence change does not change fingerprint | high | Add `impression_fingerprint` to token_payload; test determinism |
| T-65-04: False authoritative population claims | high | Keep `authoritative_population_count?: false` for all bases |
</threat_model>

<task id="65-02-01" type="execute">
  <name>Schema v2 and fingerprint token</name>
  <read_first>
    - rulestead/lib/rulestead/targeting/impact_preview.ex
    - rulestead/lib/rulestead/targeting/preview_evidence/limits.ex
    - .planning/phases/65-host-preview-evidence-contract/65-CONTEXT.md (D-04, D-05)
  </read_first>
  <action>
In `impact_preview.ex`:

1. Change `@schema_version` from `1` to `2`.

2. Add `@impression_allowlist` matching D-06 (same keys as Limits).

3. Add `redacted_impression_summary/1`:
   - Accept nil → `%{}`
   - Accept map → allowlist + normalize integers
   - Used in build and fingerprint

4. Update `preview_fingerprint/1` token_payload to include:
   ```elixir
   impression_fingerprint: fingerprint(redacted_impression_summary(fetch(attrs, :impression_summary)))
   ```
   Keep existing `sample_fingerprint`, `preview_basis`, definition fingerprints.

5. Update `build/1` output map:
   - `preview_schema_version: 2`
   - Add `impression_evidence: redacted_impression_summary(fetch(attrs, :impression_summary))`
   - Keep `sample_evidence` as today

6. Add `preview_basis/1` selection helper used by store (or document store sets basis):
   - `"authored_state_and_explicit_samples"` — default when no host impression/samples from resolver path (only command samples or empty)
   - `"authored_state_with_host_evidence"` — when merged samples non-empty OR impression_evidence non-empty map
   - `"authored_state_host_evidence_unavailable"` — when resolver was attempted but returned empty/error path (store passes `preview_basis` attr explicitly)

7. Update `@uncertainty_message` to vary by basis:
   - `authored_state_and_explicit_samples` → `"authored-state and explicit-sample preview only"`
   - `authored_state_with_host_evidence` → `"authored state with bounded host-supplied evidence; not an authoritative population count"`
   - `authored_state_host_evidence_unavailable` → `"authored-state preview; host evidence unavailable or denied"`

8. Export `redacted_impression_summary/1` if needed by store tests (`@doc false`).

**Breaking change note:** v1 fingerprints invalid after bump — expected; apply gate uses `schema_version()`.
  </action>
  <verify>
    <automated>cd rulestead && mix test test/rulestead/targeting/impact_preview_test.exs</automated>
  </verify>

    - `preview_fingerprint/1` token includes `:impression_fingerprint` key (grep `impression_fingerprint` in impact_preview.ex)
    - Built preview includes `:impression_evidence` key
    - `uncertainty.authoritative_population_count?` is `false` in build output
    - `mix test test/rulestead/targeting/impact_preview_test.exs` exits 0
  </acceptance_criteria>
</task>
