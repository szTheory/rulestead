`assemble_preview_evidence_attrs(environment_key, audience, command, affected_references, resolver_opts \\ [])`

Steps:
1. Build query map for `PreviewEvidence.Query.new/1`:
   - `environment_key`, `tenant_key` from command/audience
   - `audience_key: audience.key`
   - `operation: command.operation`
   - `before_definition`, `after_definition` (same logic as current audience_preview_payload)
   - `affected_reference_keys: AudienceDependencies.reference_keys(affected_references)`

2. `command_samples = Map.get(command, :samples, [])`

3. If `PreviewEvidence.resolver_module(resolver_opts)` is nil:
   - `merged_samples = command_samples`
   - `impression_summary = %{}`
   - `preview_basis = Map.get(command, :preview_basis) || "authored_state_and_explicit_samples"`

4. Else:
   - `{:ok, evidence} = PreviewEvidence.resolve(query, resolver_opts)` OR error → return `{:error, error}` upward (fail-closed)
   - On `{:error, _}` from resolve path with configured resolver but empty usable evidence, store may still build with basis `authored_state_host_evidence_unavailable` **only** when error is `:empty` — otherwise propagate invalid_command
   - `merged_samples = Limits.merge_samples(command_samples, Map.get(evidence, :samples, []), resolver_opts)`
   - `impression_summary = Map.get(evidence, :impression_summary, %{})`
   - Set `preview_basis`:
     - If command has explicit `preview_basis`, respect it
     - Else if samples or impression non-empty → `"authored_state_with_host_evidence"`
     - Else if resolver configured but empty → `"authored_state_host_evidence_unavailable"`
     - Else default explicit-only basis

5. Return map attrs for `ImpactPreview.build/1` including `samples`, `impression_summary`, `preview_basis`, plus existing keys.

Update `audience_preview_payload/4` in both modules to:
- Compute `affected_references` as today
- Call `assemble_preview_evidence_attrs/...`
- On `{:error, error}`, raise or return error — for preview command path return `{:error, error}`; ensure `preview_audience_impact` surfaces it

Pass `resolver_opts` from `Keyword.get(opts, :preview_evidence_resolver_opts, [])` on store calls where tests need override.
