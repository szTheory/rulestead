# Roadmap: v0.3.0 - Ecosystem Integration & Lifecycle Hygiene

## Phases

- [x] **Phase 14: OpenFeature Ecosystem Integration** - Implement an official `open_feature_rulestead` provider to align with CNCF standards.
- [ ] **Phase 15: Lifecycle Hygiene & Code References** - Build tools to detect stale flags, find code references, and automate cleanup visibility.

## Phase Details

### Phase 14: OpenFeature Ecosystem Integration
**Goal**: Operators can adopt Rulestead without vendor lock-in by using the standard OpenFeature API for evaluation.
**Depends on**: None
**Requirements**: ECO-01, ECO-02, ECO-03
**Success Criteria** (what must be TRUE):
  1. Rulestead provides an Elixir OpenFeature provider conforming to the OpenFeature spec.
  2. Host applications can evaluate Rulestead flags via the generic OpenFeature client and receive accurate variant results.
  3. Rulestead's unique explainability trace is safely translated and surfaced in OpenFeature's standard evaluation details.
**Plans**: 14-01, 14-02

### Phase 15: Lifecycle Hygiene & Code References
**Goal**: Operators can confidently identify and remove obsolete flags to eliminate technical debt.
**Depends on**: Phase 14
**Requirements**: LCH-01, LCH-02, LCH-03
**Success Criteria** (what must be TRUE):
  1. User can view a dedicated dashboard or view in the Admin UI showing flags that are candidates for removal (stale flags).
  2. The system can ingest and store code references for flags from an external repository scanner (e.g., GitHub Action).
  3. User can navigate from a stale flag in the Admin UI directly to its exact code usage locations.
**Plans:** 3 plans
- [x] 15-01-PLAN.md — Stale Flag Telemetry Cache
- [x] 15-02-PLAN.md — Code Reference Scanner & Ingress
- [ ] 15-03-PLAN.md — Admin UI Cleanup Workflow
