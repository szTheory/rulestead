Create `rulestead/lib/rulestead/governance/rollout_auto_advance.ex`:

**Public API:**
```elixir
def automation_tick?(metadata) when is_map(metadata)
def execute_scheduled_tick(store, scheduled_execution, execute_command)
```

**automation_tick?/1** — true when `metadata["source"] == "guardrail_automation"` OR `metadata[:source] == :guardrail_automation`.

**execute_scheduled_tick/3 pipeline:**

1. `validate_snapshot_freshness/2` — compare snapshot `"rollout"."stage"`, `"percentage"`, `"monitoring_window_ends_at"` against latest GuardrailDecision + active ruleset for resource. Mismatch → `{:error, StoreError.invalid_command("rollout_stage_conflict")}` or new reason `"auto_advance_superseded"`.

2. `fetch_enabled_policy/3` — store `fetch_rollout_auto_advance_policy/1`; must be enabled + complete or return blocked outcome.

3. `resolve_signal_facts/3` — load guardrail signal keys from active rollout rule config; for each key call `Rulestead.Guardrails.fetch_signal/2`; build signal_facts list.

4. `evaluate_rollout_auto_advance/1` via store with:
   - `monitoring_window_ends_at` from snapshot
   - `evaluated_at: DateTime.utc_now() |> DateTime.truncate(:second)`
   - resolved signal_facts

5. On `status: :blocked` → `{:ok, %{outcome: :blocked, eligibility: eligibility, reasons: eligibility.reasons}}`

6. On `status: :eligible` → `build_advance_command/4`:
   - `stage: policy.next_stage`
   - `percentage: policy.next_percentage`
   - `monitoring_window_started_at: evaluated_at`
   - `monitoring_window_ends_at: DateTime.add(evaluated_at, policy.observation_window_seconds, :second)`
   - `signal_facts: resolved_facts`
   - `metadata: %{"source" => "guardrail_automation", "scheduled_execution_id" => scheduled_execution.id, "eligibility" => eligibility_snapshot}`
   - Validate policy next_stage still exists in ruleset; mismatch → `{:error, "auto_advance_ruleset_conflict"}`

Module accepts store module as first arg for Fake/Ecto delegation pattern.

No I/O beyond store callbacks and Guardrails.fetch_signal.

In `execute_direct_scheduled_action("advance_rollout", scheduled_execution, command)`:

```elixir
defp execute_direct_scheduled_action("advance_rollout", scheduled_execution, command) do
  if RolloutAutoAdvance.automation_tick?(scheduled_execution.metadata) do
    case RolloutAutoAdvance.execute_scheduled_tick(__MODULE__, scheduled_execution, command) do
      {:ok, %{outcome: :blocked} = blocked_result} ->
        {:ok, blocked_result}

      {:ok, %{outcome: :change_request_submitted} = cr_result} ->
        {:ok, cr_result}

      {:ok, advance_command} when is_struct(advance_command, Command.AdvanceRollout) ->
        advance_rollout(advance_command)

      {:error, reason} ->
        {:error, reason}
    end
  else
    # existing direct snapshot replay path unchanged
    advance_rollout(Command.AdvanceRollout.new(...))
  end
end
```

Ensure `run_scheduled_execution/3` passes orchestrator `{:ok, %{outcome: :blocked}}` to `finalize_scheduled_execution_success/4` with execution_metadata containing eligibility reasons — NOT failure finalize.

On successful advance via orchestrator, audit from `advance_rollout/1` must include metadata `"source" => "guardrail_automation"` on rollout.advance event (via advance command metadata merge in audit_event_changeset path).
