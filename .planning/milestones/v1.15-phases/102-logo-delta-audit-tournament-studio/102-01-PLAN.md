Create `scripts/gen_glyph_paths.py` that generalizes `scripts/gen_wordmark_paths.py` with the following contract. Do NOT import urllib anywhere in the file.

CLI interface:
  --font-url URL     Required. Must start with https://fonts.gstatic.com/ (assert enforced per T-97-03)
  --text TEXT        Default "Rulestead"
  --em-size FLOAT    Default 64.0 (SVG units for the em height)
  --tracking FLOAT   Default 0.0 (em fraction per glyph; -0.02 = tighter by 2% em per glyph)
  --weight LABEL     Optional display label only (e.g. "700"); printed in stderr header

Font download: Use `subprocess.run(["curl", "-s", "-o", out_path, url], capture_output=True, timeout=30)`. Write to a `tempfile.mkdtemp(prefix="rulestead_glyphs_")` directory. Clean up the temp dir in a `finally` block via `shutil.rmtree`.

sys.path: Insert `os.path.expanduser('~/Library/Python/3.14/lib/python/site-packages')` at index 0 before any fontTools import (mirrors gen_wordmark_paths.py line 32; needed for user-installed fontTools 4.62.1).

Per-glyph emission (CRITICAL — must NOT merge paths):
- One `<path>` element per input character, each with its own `transform` attribute.
- Transform formula: `translate({x_px:.3f},{em_size:.3f}) scale({scale:.6f},-{scale:.6f})` where `scale = em_size / upm` and `x_px = x_cursor * scale`.
- Tracking: `tracking_units = tracking * upm`; add `tracking_units` to `x_cursor` after each glyph (including missing glyphs — advance by space width fallback).
- Emit a comment `  <!-- glyph: {ch} -->` on the line before each `<path>` element.
- Missing glyph (not in cmap): print warning to stderr, advance by `font["hmtx"].metrics.get("space", (500, 0))[0] + tracking_units`, skip path emission.

stdout output:
```
<!-- gen_glyph_paths.py: 'Rulestead' em=64.0 tracking=0.0 -->
<!-- Font: {url} -->
<!-- Total advance width: {w:.3f} SVG units -->
  <!-- glyph: R -->
  <path transform="translate(0.000,64.000) scale(0.064000,-0.064000)" d="..."/>
  <!-- glyph: u -->
  <path transform="translate(42.048,64.000) scale(0.064000,-0.064000)" d="..."/>
  ...
```
