<objective>
Fix two evidence-based IA clarity gaps found in the real both-theme renders:

1. The Overview (home) rail link uses a modifier class (`rs-shell__rail-link--overview`) that has no CSS rules — it looks identical to every grouped nav item and provides no orientation anchor for the operator/SRE scanning the rail at a glance.

2. The "Needs you now" empty state (`.rs-attention-empty`) uses `--rs-surface-faint` as its background. In dark mode this token resolves to `#10161f` — the absolute darkest surface in the palette, darker than `--rs-bg` (`#19222e`). The "nothing to action" card reads as a sunken void rather than a calm green-pass signal.

Purpose: IA-01 (home makes "what needs me" obvious at a glance) and IA-02 (rail orientation consistent across screens, both themes).

Output: Two targeted CSS additions + one token swap. No structural changes to shell.ex markup or navigation.ex.
</objective>

    Immediately after the .rs-shell__rail-link[aria-current="page"] block (around line 1538),
    add a CSS block for .rs-shell__rail-link--overview.

    The goal is visual separation between the Overview anchor and the task-rhythm groups
    below it, so operators locate "home" instantly on the rail.

    Rules to add:

    /* Overview anchor — visually distinct from task-rhythm group links */
    .rs-shell__rail-link--overview {
      font-weight: var(--rs-weight-semibold);
      margin-bottom: var(--rs-space-2);
      padding-bottom: var(--rs-space-3);
      border-bottom: 1px solid var(--rs-border-subtle);
    }

    On mobile (horizontal scroll bar, < 48rem) the border-bottom would be
    awkward inside a flex row — suppress it at mobile:

    @media (max-width: 47.99rem) {
      .rs-shell__rail-link--overview {
        border-bottom: none;
        margin-bottom: 0;
        padding-bottom: 0.3rem;
      }
    }

    Rationale: semibold weight + a hairline bottom separator (using the existing
    --rs-border-subtle token) creates a clear visual break between "where I am"
    and "where I can go" without adding color or a new sub-AA pair. The separator
    uses --rs-border-subtle, which is token-driven and re-themes correctly.
    On mobile the rail is a horizontal pill strip, so the separator is suppressed
    to avoid misreading as a focus underline.

    Do NOT add background color or icon — the existing [aria-current="page"]
    rule already handles the "currently on home" state with --rs-primary-soft.
  </action>
