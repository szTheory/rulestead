The fix: add a new `data-theme-switching` attribute to the existing suppression CSS selector, then have the `.ThemeControl` hook set that attribute immediately before every `applyTheme()` call and remove it on the next `requestAnimationFrame`.

Purpose: Confirm MOT-01 principle-alignment and eliminate the only observable motion defect (theme-toggle flicker). Bias to NOT adding motion.

    3. Removes `data-theme-switching` on the NEXT `requestAnimationFrame` (after the browser has committed the frame with no transitions).

    The cleanest implementation is to modify `applyTheme` itself, so the suppression is automatic for every caller (click, keyboard, and any future path):

    Current `applyTheme`:
      const applyTheme = (val) => {
        this._mode = val
        if (val === "dark")       shell.setAttribute("data-theme", "dark")
        else if (val === "light") shell.setAttribute("data-theme", "light")
        else                      shell.removeAttribute("data-theme")
      }

    Updated `applyTheme`:
      const applyTheme = (val) => {
        shell.setAttribute("data-theme-switching", "")
        this._mode = val
        if (val === "dark")       shell.setAttribute("data-theme", "dark")
        else if (val === "light") shell.setAttribute("data-theme", "light")
        else                      shell.removeAttribute("data-theme")
        requestAnimationFrame(() => shell.removeAttribute("data-theme-switching"))
      }

    IMPORTANT — the mount path calls `applyTheme(readTheme())` immediately followed by `shell.removeAttribute("data-theme-pending")`. With the updated `applyTheme`, the mount path will set `data-theme-switching` then schedule a rAF to remove it. This is harmless: `data-theme-pending` is still removed synchronously (the existing line 380 is untouched), and the `data-theme-switching` attribute disappears after the first paint frame. The Phase 90 "no animated wipe on initial load" guarantee is unaffected because `data-theme-pending` still covers the initial snap, and `data-theme-switching` is gone within a frame anyway.

    Do NOT change `updated()`, `destroyed()`, or any other logic in the hook. This is an additive change to one function only.
