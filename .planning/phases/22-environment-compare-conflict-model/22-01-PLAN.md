<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authored store records -> compare payload | Stored authored state crosses into an operator-visible compare contract here |
| compare fingerprint inputs -> compare token | Scoped authored facts are compressed into a stale-preview contract here |
| fake parity seam -> contract tests | Test-only adapter state is asserted against the real compare contract here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01-01 | T | compare token scope | mitigate | Fingerprint only the compared flag set plus dependency closure per D-15 to D-17, and include schema version in the token so unrelated churn cannot force false-stale previews. |
| T-22-01-02 | I | compare payload | mitigate | Keep authored compare limited to flag metadata, published rulesets, dependency closure, and explicit warnings; do not leak runtime snapshots, approval internals, or raw session metadata per D-08 and D-09. |
| T-22-01-03 | R | stale-preview truthfulness | mitigate | Emit explicit blocker findings for source-changed, target-changed, and stale-token conflicts rather than silently reusing old previews. |
| T-22-01-04 | D | compare query path | mitigate | Reuse bounded projection helpers and scoped dependency traversal instead of whole-environment raw diffing, and test only differing/problem flags in the result set to avoid unnecessary expansion. |
</threat_model>
