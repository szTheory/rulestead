## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| governance event emission -> outbound product state | internal high-impact events become deliverable external notifications here |
| delivery worker -> remote destination | signed outbound HTTP requests leave the host boundary here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-10 | I | outbound signer | mitigate | Keep destination secrets out of audit, telemetry, and UI payloads; sign deliveries through bounded helper inputs or redacted references only per D-16. |
| T-12-11 | T | delivery retry state | mitigate | Persist one durable delivery identity and mutate retry state transactionally so repeated worker execution cannot create silent duplicate product truth per D-15. |
| T-12-12 | D | exhausted outbound retries | mitigate | Stop in an operator-visible terminal state after bounded retries and require explicit retry action instead of silent infinite background retries per D-15. |
</threat_model>
