| verifier outcome -> durable receipt persistence | security-critical rejection reasons and accepted envelopes become operator-visible product truth here |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | S | `Rulestead.Webhooks.Verifier` | mitigate | Validate signatures with provider adapters, constant-time comparison, endpoint-scoped secrets, and explicit freshness windows per D-01, D-02, and D-04. |
| T-12-02 | R | receipt persistence | mitigate | Persist every accepted and rejected ingress attempt with stable delivery identity, verification metadata, and rejection reason so operators can audit why the mutation path did or did not run per D-03 and D-04. |
| T-12-03 | D | replay handling | mitigate | Add replay-claim persistence and dedupe keys so duplicate or retried deliveries fail closed before mutation execution while preserving the first durable receipt per D-03 and D-04. |
| T-12-04 | T | HTTP ingress Plug | mitigate | Capture the raw body at the library-owned Plug boundary, return `401` on malformed or failed verification, halt the connection, and prove in HTTP tests that no governance mutation facade runs on rejection per D-01, D-02, and D-04. |
</threat_model>
