## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| maintainer local env (`HEX_API_KEY`) → Hex registry | The irreversible publish authenticates with the local-only API key; never committed or logged. |
| provider `mix.lock` → published tarball | The dry-run dependency-list inspection is the human boundary check that `rulestead` ships as a hex dep, not a silently-dropped path dep. |
| git tag → HexDocs [source] links / tarball provenance | The `open_feature_rulestead-v1.0.0` tag binds the published source to a verifiable commit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-129-07 | Tampering | silent path-dep drop at publish | mitigate | Guard (129-02) + the blocking dry-run dependency-list human confirmation in Task 1 before the irreversible publish (D-14). |
| T-129-08 | Information Disclosure | `HEX_API_KEY` at publish time | mitigate | Key supplied via local env only at the `mix hex.publish --yes` checkpoint; never written to repo, logs, or the verification doc. |
| T-129-09 | Spoofing/Repudiation | published-source provenance | mitigate | `open_feature_rulestead-v1.0.0` tag at the publish commit + `source_ref` (set in 129-01) make HexDocs [source] links and tarball origin verifiable. |
| T-129-10 | Denial of Service | publishing before core is live | mitigate | Blocking checkpoint + Task 1 `curl` precondition refuse to publish unless `rulestead@1.0.0` returns 200. |
| T-129-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs; `mix deps.get` resolves already-audited Hex deps only. |
