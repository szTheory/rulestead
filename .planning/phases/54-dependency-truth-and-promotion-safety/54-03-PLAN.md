<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Compare/export artifacts -> apply/import execution | Saved plans/manifests may become stale relative to live dependency truth. |
| Source/target environments -> promotion scope mapping | Cross-scope ambiguity can hide tenant/environment mismatches. |
| Dependency findings -> operator/support decisions | Unclear or unstable findings can cause unsafe promotion/import actions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-13 | Tampering | Saved compare/replay token reuse | mitigate | Revalidate live dependency truth at apply/re-apply time and block stale/incompatible plans. |
| T-54-14 | Elevation of privilege | Cross-tenant promotion/import | mitigate | Require explicit environment/tenant scope on all findings and block mismatches. |
| T-54-15 | Repudiation | Promotion/manifest blocker ambiguity | mitigate | Emit stable, code-based findings in deterministic order for support reconstruction. |
| T-54-16 | Information Disclosure | Manifest finding details | mitigate | Apply existing redaction rules for unauthorized resources while preserving actionable blocker counts. |
| T-54-17 | Denial of Service | Large finding sets | mitigate | Use bounded pagination/batching and deterministic sorting with stable keys. |
| T-54-18 | Tampering | Adapter behavior drift | mitigate | Keep Ecto/Fake contract tests for compare/apply/import blocker behavior in lockstep. |
</threat_model>
