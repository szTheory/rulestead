<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin caller -> dependency inventory API | Untrusted read parameters cross authorization/redaction boundaries. |
| Projection refresh -> persisted dependency truth | Incorrect projection updates could mislead mutation and promotion safety decisions. |
| Authorized + unauthorized resources -> response envelope | Mixed visibility must not leak sensitive resource identity while preserving useful truth. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-54-01 | Information Disclosure | Inventory read response | mitigate | Enforce policy action checks and redacted partial truth (`hidden_reference_count` + sanitized placeholders). |
| T-54-02 | Tampering | Projection identity collisions | mitigate | Key projection rows by full semantic identity including `environment_key` and `tenant_key`. |
| T-54-03 | Repudiation | Inventory mismatch across adapters | mitigate | Add Ecto/Fake parity contract tests for identical ordering and scope fields. |
| T-54-04 | Elevation of privilege | Cross-scope confusion | mitigate | Require explicit scope in every entry and in aggregate response metadata. |
| T-54-05 | Denial of Service | Expensive ad hoc scans | mitigate | Read from indexed projection with bounded pagination and deterministic ordering. |
| T-54-06 | Tampering | Runtime purity drift | mitigate | Keep inventory/projection code out of evaluator/runtime snapshot hot path and assert no runtime dependency calls in tests. |
</threat_model>
