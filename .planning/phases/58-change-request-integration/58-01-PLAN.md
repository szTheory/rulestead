# Plan 58-01: Audience Mutation Change Request Contract

<objective>
Extend governance vocabulary with `:apply_audience_mutation` and implement pure submit/embedding helpers plus `governed_apply?` threshold bypass in `BlastRadiusThreshold`.
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-58-01: Submit below-threshold via CR to skip review | high | `validate_submit` rejects verdict != :above_threshold |
| T-58-02: Execute with stale preview after approval | high | Execute path still calls ensure_fresh_audience_preview (plan 58-02) |
| T-58-03: Bypass threshold on indeterminate via governed_apply | high | governed_apply? only skips :above_threshold, not :indeterminate |
</threat_model>

  <action>
Create `rulestead/lib/rulestead/governance/audience_mutation_change_request.ex`:

**Module:** `Rulestead.Governance.AudienceMutationChangeRequest`

**Public API:**
- `@spec validate_submit(Command.SubmitChangeRequest.t(), map()) :: :ok | {:error, Rulestead.Error.t()}`
  - Second arg `current_preview` map from store (same shape as `audience_preview_payload`)
- `@spec build_submission_metadata(map(), map()) :: map()`
  - Args: `assessment` (from BlastRadiusThreshold.assess), `current_preview`
  - Returns metadata map per CONTEXT D-02 with string keys

**validate_submit/2 rules:**
- `command.action == :apply_audience_mutation`
- `Compare.protected_target?(command.environment_key)` — else invalid_command "audience change requests require a protected environment"
- Run `BlastRadiusThreshold.assess/2` from command + preview inputs; require `verdict == :above_threshold`
- Reject `:below_threshold` with remediation "Direct apply is allowed for this blast radius; do not submit a change request."
- Reject `:indeterminate` with remediation from assessment breach_reasons
- `command.preview_fingerprint` must equal `current_preview.preview_fingerprint` (or `Map.get(current_preview, :preview_fingerprint)`)
- `command.command` map must include required ApplyAudienceMutation keys: `audience_key`, `operation`, `preview_schema_version`, `preview_fingerprint`

**build_submission_metadata/2:**
- Include `"blast_radius_assessment"` => Map.drop(assessment, []) with string keys via existing normalize patterns
- Include `"affected_reference_summary"` with reference_count, distinct_flag_count, reference_keys, rollout_hints, lifecycle_hints from preview

Use `StoreError.invalid_command/2` for all failures.
