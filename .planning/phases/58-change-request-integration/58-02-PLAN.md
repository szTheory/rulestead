# Plan 58-02: Fake Store Change Request Integration

<objective>
Integrate audience mutation change requests in `Rulestead.Fake`: submit validation with embedded metadata, governed execute path, and contract tests (Fake-only section).
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-58-05: Direct apply after submit without approval | medium | execute requires approved state (existing ensure_governance_transition) |
| T-58-06: Tenant scope drift between submit and execute | medium | Revalidate tenant from command_snapshot on execute |
</threat_model>

<tasks>

<task id="58-02-01" type="execute">
  <name>Submit hook for audience mutation CRs</name>
  <read_first>
    - rulestead/lib/rulestead/fake.ex

    - rulestead/lib/rulestead/fake.ex
    - search within file: handle_call submit_change_request
  </read_first>
  <action>
In `fake.ex` `handle_call({:submit_change_request, command}, ...)`:

Before building `change_request` map, when `command.action == :apply_audience_mutation`:

1. Build `current_preview` using same helper as `do_apply_audience_mutation` (`audience_preview_payload/4`) from `command.command` fields (cast to ApplyAudienceMutation or read map keys).

2. Call `AudienceMutationChangeRequest.validate_submit(command, current_preview)`.

3. Merge `AudienceMutationChangeRequest.build_submission_metadata(assessment, current_preview)` into `command.metadata` (assessment from re-assess or returned by validate — prefer assess inside build after validate passes).

4. Set `resource_type: "audience"`, `resource_key` from `command.command["audience_key"]`.

On validation error, return `{:reply, {:error, error}, state}` without inserting change request.
