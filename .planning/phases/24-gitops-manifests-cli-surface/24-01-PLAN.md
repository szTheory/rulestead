## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01-01 | I | manifest export payload | mitigate | Export only authored-state fields per D-05 and explicitly exclude IDs, timestamps, runtime snapshots, audit/governance state, operational overrides, drafts, and compare metadata per D-06. |
| T-24-01-02 | T | canonical serializer | mitigate | Canonicalize nested structures and sort flags, dependency keys, and semantic lists before encoding so manifest bytes do not drift from incidental map order. |
| T-24-01-03 | R | parser/serializer contract | mitigate | Route export and downstream consumers through one shared `load` plus `serialize` contract so validate, diff, and import cannot silently reinterpret manifest bytes differently. |
| T-24-01-04 | D | adapter-specific export query path | mitigate | Prove Ecto/Fake parity with one shared contract test so no adapter leaks a divergent field set or ordering contract. |
| T-24-01-05 | E | task argument handling | mitigate | Require explicit environment scope, use strict option parsing, and treat unexpected usage as CLI failure rather than silently exporting the wrong environment. |
</threat_model>
