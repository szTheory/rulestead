| Boundary | Description |
|----------|-------------|
| host-owned actor/policy seam -> governance contracts | Governance structs receive privileged actor and policy facts from the host boundary here |
| admin/operator intent -> change-request payload snapshot | High-impact mutation intent crosses into durable governance state here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01-01 | S | governance actor fields | mitigate | Keep actor data as explicit host-supplied fields only; do not add bundled auth or session-derived identity per Phase 7 D-29. |
| T-09-01-02 | T | change-request command snapshot | mitigate | Require explicit governed action, target resource, environment key, and correlation id in the contract instead of inferring them from later UI or scheduler state. |
| T-09-01-03 | R | approval/self-approval semantics | mitigate | Encode required approval count and self-approval allowance as explicit contract data so later policy enforcement is auditable and testable. |
| T-09-01-04 | I | governance contract metadata | mitigate | Exclude raw session/request objects and keep only redacted, structured command metadata in the domain contract per Phase 7 D-33 and D-34. |
</threat_model>
