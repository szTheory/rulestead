<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host policy module -> authorizer | External application code supplies the final governance policy decision here |
| approver actor -> approval decision | A potentially privileged actor attempts to approve a governed mutation here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-03-01 | S | self-approval path | mitigate | Default-deny production self-approval in the authorizer unless the host explicitly opts in via `allow_self_approval?`. |
| T-09-03-02 | T | direct prod mutation path | mitigate | Resolve `change_request_required?` before execution so production publish/rollout/kill-switch/settings actions can be converted into governed requests. |
| T-09-03-03 | E | policy callback failures | mitigate | Fail closed on missing/erroring security-critical governance hooks rather than falling through to permissive execution. |
| T-09-03-04 | R | approval posture drift | mitigate | Emit explicit approval-requirement snapshots that later plans persist and audit, instead of recomputing opaque policy state after the fact. |
</threat_model>
