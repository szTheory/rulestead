| Boundary | Description |
|----------|-------------|
| host app provider -> `Rulestead.Guardrails.Provider` | Untrusted provider-specific data crosses into core contract code and must be normalized before later rollout logic can consume it |
| query builder -> signal fact normalizer | Missing scope or weak evidence could otherwise be mistaken for healthy rollout data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-49-01 | T | `Rulestead.Guardrails.SignalFact` | mitigate | Normalize every provider outcome into a bounded status/reason vocabulary and reject freeform provider truth as stable contract data. |
| T-49-02 | I | `Rulestead.Guardrails.SignalQuery` | mitigate | Require explicit environment scope and optional tenant scope in the query struct; do not infer from runtime/session state. |
| T-49-03 | E | `Rulestead.Guardrails.Provider` | mitigate | Keep the seam behaviour-only and host-owned so core code never gains hidden provider credentials or direct observability access. |
</threat_model>
