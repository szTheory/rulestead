# Phase 18: Experimentation UI & Reporting

## Goal
Product managers and operators can view the results of A/B tests and monitor for regressions in the Admin UI.

## Depends on
Phase 16, Phase 17

## Requirements
- **ANA-03**: Calculate and expose experiment results (e.g., conversion lift, statistical significance) and guardrail metrics (e.g., unexpected error rates) in the Admin UI.

## Success Criteria (what must be TRUE)
1. The Admin UI has a dedicated section for viewing active and historical experiments.
2. Operators can see calculated conversion lifts and statistical significance for active experiments.
3. The UI warns operators of unexpected error rates (guardrail metrics) associated with specific variations.
