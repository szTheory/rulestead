    --- `examples/demo/frontend/tests/support/contrast-check.ts` ---

    Export a single function `wcagRatio(hex1: string, hex2: string): number` that:
    1. Parses two hex color strings (6-char, no alpha channel — the token hex values from RESEARCH.md §3 are all opaque).
    2. Computes relative luminance for each using the W3C formula: convert each channel to 0–1, linearize via `c <= 0.04045 ? c/12.92 : ((c+0.055)/1.055)^2.4`, then L = 0.2126*R + 0.7152*G + 0.0722*B.
    3. Returns `(Lmax + 0.05) / (Lmin + 0.05)`.

    Also export `assertAA(ratio: number, level: 'normal' | 'large'): void` that throws if ratio < 4.5 (normal) or < 3.0 (large).

    --- `examples/demo/frontend/tests/theme-cascade.spec.ts` ---

    Import `{ test, expect }` from `@playwright/test`. Import `path` and build the harness URL via `'file://' + path.resolve(__dirname, '../../../../rulestead_admin/priv/static/theme-harness.html')`.

    Write 5 tests matching the cascade-precedence matrix from VALIDATION.md:

    Test 1 "no attr, light OS — light tokens active": use `browser.newPage` with `colorScheme: 'light'`, goto harness, assert `--rs-surface` resolves to `#ffffff` (the light value).

    Test 2 "no attr, dark OS — dark tokens active (THM-01)": use `colorScheme: 'dark'`, goto harness with no data-theme, assert `--rs-neutral-0` resolves to a value starting with `#1` (mineral dark range, not `#fff`). Also assert `--rs-surface` is NOT `#ffffff`.

    Test 3 "pinned dark, light OS (THM-03)": use `colorScheme: 'light'`, goto harness, call `page.evaluate(() => window.setTheme('dark'))`, assert `--rs-neutral-0` does NOT equal `#ffffff`.

    Test 4 "pinned light, dark OS (THM-03)": use `colorScheme: 'dark'`, goto harness, call `page.evaluate(() => window.setTheme('light'))`, assert `--rs-surface` equals `#ffffff`.

    Test 5 "pinned dark, dark OS": use `colorScheme: 'dark'`, goto harness, call `page.evaluate(() => window.setTheme('dark'))`, assert `--rs-neutral-0` does NOT equal `#ffffff`.

    For all token value assertions, evaluate via:
    ```
    page.evaluate(() =>
      getComputedStyle(document.querySelector('.rs-shell')).getPropertyValue('--rs-neutral-0').trim()
    )
    ```

    Each test uses `browser.newContext({ colorScheme: ... })` + `context.newPage()` to set OS emulation.

Since the fallback is `red` = `rgb(255, 0, 0)`, assert it equals `rgb(255, 0, 0)`. Note: this test will only be meaningful after Plan 02 moves --rs-bg off :root; mark with a comment.
