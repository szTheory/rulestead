<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Store command -> Ecto transaction | Authorized mutation request becomes durable DB change only after preview revalidation. |
| Ecto transaction -> audit ledger | Security-sensitive evidence is persisted for support reconstruction. |
| Authored DB rows -> runtime snapshot | Stored audience definitions become compiled runtime data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-19 | Tampering | Stale preview replay | mitigate | Rebuild current preview inside `Ecto.Multi` and compare command fingerprint before mutation. |
| T-53-20 | Elevation of privilege | Cross-tenant/cross-environment preview confirmation | mitigate | Fetch scoped environment/audience and reject tenant/environment mismatches before update/archive. |
| T-53-21 | Information Disclosure | Raw sample evidence/PII leakage | mitigate | Persist only `ImpactPreview` redacted `sample_evidence`; extend `AuditEvent.metadata/1` scrubber for preview keys. |
| T-53-22 | Tampering | Runtime live lookup drift | mitigate | Snapshot payload includes compiled audience definitions; evaluator consumes snapshot data from Plan 02. |
| T-53-23 | Repudiation | Audit evidence tampering/omission | mitigate | Mutation and audit insert happen in one `Ecto.Multi`; mutation rolls back if audit insert fails. Existing append-only audit controls remain unchanged. |
| T-53-24 | Elevation of privilege | Protected shared-targeting mutation bypass | mitigate | Protected/shared mutation attempts require fresh fingerprint and fail closed with audit evidence when missing/stale. |
</threat_model>
