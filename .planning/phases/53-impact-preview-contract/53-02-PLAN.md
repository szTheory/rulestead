<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Persisted snapshot payload -> runtime compiler | Stored binary payload becomes trusted in-memory evaluation data only after validation. |
| Runtime context -> evaluator | Host-supplied context traits influence audience matching. |
| Audience reference -> evaluator trace | Missing/archived audience state becomes support-visible trace evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-07 | Tampering | Runtime live lookup drift | mitigate | Compile `audiences` into `Runtime.Snapshot` and reject evaluator calls to Store/Repo/Admin/Telemetry/observability/audit for audience resolution. |
| T-53-08 | Information Disclosure | Evaluator debug trace | mitigate | Trace only audience key and match/miss/missing/archived reason; never include raw context or sample evidence. |
| T-53-09 | Denial of Service | Malformed snapshot payload | mitigate | Reject non-map audience definitions and blank keys through existing malformed runtime data error path. |
| T-53-10 | Repudiation | Missing audience behavior | mitigate | Emit deterministic trace reasons for missing/archived audience references so support can explain outcomes. |
| T-53-11 | Elevation of privilege | Cross-tenant/cross-environment runtime reference | mitigate | Runtime only consumes the compiled snapshot for one environment; Plan 04 builds scoped snapshot payloads. |
| T-53-12 | Tampering | Protected shared-targeting bypass | mitigate | Archived/missing compiled audiences skip/fail closed rather than silently matching. |
</threat_model>
