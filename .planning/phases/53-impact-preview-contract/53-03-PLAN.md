## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Public API caller -> `Rulestead` facade | Untrusted command data enters admin read/write authorization envelope. |
| Facade -> Store behavior | Authorized commands cross into adapter implementations. |
| Fake/Redis adapters -> contract tests | Non-Ecto adapters must not weaken fail-closed semantics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-53-13 | Tampering | Stale preview replay | mitigate | `ApplyAudienceMutation` requires `preview_schema_version` and `preview_fingerprint`; Fake rebuilds current preview before mutation. |
| T-53-14 | Elevation of privilege | Cross-tenant/cross-environment confirmation | mitigate | Command structs and resources carry explicit `environment_key` and `tenant_key`; mismatches fail closed. |
| T-53-15 | Information Disclosure | Raw sample evidence | mitigate | Public path receives samples but Plan 01 redaction contract controls preview output; tests assert sensitive keys are absent. |
| T-53-16 | Tampering | Runtime live lookup drift | mitigate | This plan only wires admin command surfaces; runtime lookup proof remains Plan 02 and no evaluator changes are made here. |
| T-53-17 | Repudiation | Audit evidence omission | mitigate | Command fields require actor/reason/metadata/scope so Plan 04 can persist reconstructable audit events. |
| T-53-18 | Elevation of privilege | Protected shared-targeting mutation bypass | mitigate | `protected_shared_targeting?` and preview fingerprint are command-level inputs; Fake fails closed when confirmation is missing/stale. |
