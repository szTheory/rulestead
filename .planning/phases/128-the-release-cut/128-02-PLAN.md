<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| release PR → main branch | Squash-merge is irreversible once done; version tags are created immediately |
| hex-publish environment gate → HEX_API_KEY | Key is only exposed to publish-core and publish-admin; never to preflight or approval steps |
| publish pipeline → hex.pm | Publish jobs have idempotency guards (curl check before publishing); a second run will skip, not duplicate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-128-05 | Tampering | Merge before CI is green on PR branch | mitigate | Task 1 explicitly requires human to verify all checks green before squash-merging |
| T-128-06 | Denial of Service | gate-ci-green polling timeout (30×30s) | mitigate | Manual gh workflow run ci.yml --ref rulestead-v1.0.0 dispatch documented in task action |
| T-128-07 | Information Disclosure | HEX_API_KEY exposed in wrong job | accept | Workflow design scopes key to publish-core and publish-admin only; already verified in RESEARCH.md |
| T-128-08 | Repudiation | Publish happened without human approval | mitigate | hex-publish environment requires explicit "Approve and deploy" click; blocking human gate in Task 2 |
| T-128-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs in this phase |
</threat_model>
