<objective>
Pre-cut gate, disable auto-merge, add release-as, open the release PR, hand-add the CHANGELOG
preamble to both packages, and eyeball the full diff before the irreversible hand-merge.

Purpose: The entire irreversibility risk of the 1.0.0 cut is front-loaded here. Every step in
this plan is ordered to prevent an accidental auto-merge or an incomplete diff landing in the
published tarball. The plan is autonomous: false because three of the five tasks require direct
human action in the GitHub UI or on the release PR branch.

- release-please-config.json with "release-as": "1.0.0" in the rulestead block
- The release PR open on release-please--branches--main with correct diff + preamble
- Human sign-off that the full diff is clean before any merge happens
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local workstation → main branch | All config edits in this plan are committed to main; irreversible once the release PR is auto-confirmed |
| main branch → release-please PR | release-please reads config on push; the PR is the point of no return for version numbers |
| human → release PR diff | Human is the only gate preventing an incomplete or incorrect diff from being merged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-128-01 | Tampering | release-please-config.json (release-as key added to wrong block) | mitigate | RESEARCH.md explicitly warns against adding to rulestead_admin block; task action prohibits it; grep verify confirms single match |
| T-128-02 | Tampering | release-pr-automerge workflow fires before human eyeball | mitigate | Task 2 is a blocking human gate; Task 3 is ordered strictly after; no commit to main until Task 2 confirms disabled |
| T-128-03 | Information Disclosure | HEX_API_KEY exposed in publish jobs | accept | Key is scoped to publish-core and publish-admin jobs only; not available to preflight/gate-ci-green/approval per workflow design |
| T-128-04 | Elevation of Privilege | release PR merged without diff review | mitigate | Task 4 blocking human gate; CI must be green on branch before proceed signal |
| T-128-SC | Tampering | npm/pip/cargo installs | accept | No package manager installs in this phase — pure config edits and human gate steps |
</threat_model>
