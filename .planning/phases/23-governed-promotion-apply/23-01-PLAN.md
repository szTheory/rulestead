<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| compare preview -> apply command | Operator-reviewed compare context crosses into a mutating command here |
| authored target state -> runtime snapshot | Control-plane writes cross into the runtime-consumed snapshot artifact here |
| store adapter -> immutable environment history | The applied authored state is serialized into a persisted re-apply source artifact here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01-01 | T | promotion apply bundle | mitigate | Re-run compare and fingerprint validation from the supplied compare token immediately before mutation, and reject on stale token, blocker findings, or dependency drift per D-09. |
| T-23-01-02 | I | environment-version artifact | mitigate | Persist authored configuration only per D-06; do not serialize runtime counters, telemetry artifacts, kill-switch runtime state, or hidden session data into the immutable version record. |
| T-23-01-03 | R | direct apply auditability | mitigate | Return structured compare token, flag scope, environment version id, and snapshot version so later governed/audit surfaces can correlate one exact applied bundle. |
| T-23-01-04 | D | snapshot regeneration path | mitigate | Regenerate the target runtime snapshot once per bounded bundle within the same transaction, not per flag, to avoid repeated expensive rebuilds and partial success windows. |
| T-23-01-05 | E | protected target apply | mitigate | Short-circuit protected targets into governance-required errors in this plan so direct apply cannot bypass PROM-04 or D-11. |
</threat_model>
