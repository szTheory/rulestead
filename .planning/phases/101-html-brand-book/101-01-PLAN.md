<threat_model>
ASVS level 1 posture is sufficient because this is a static, source-controlled artifact
with no server runtime and no user-submitted data.

Threats and mitigations:
- Source-truth drift: `brandbook/index.html` could become a second source of truth. Mitigate by generating text, token values, and asset previews from `brandbook/brand-book.md`, `tokens.json`, `tokens.css`, `VOICE.md`, `COPY.md`, and committed SVG files only; no hand-maintained palette or voice tables in HTML.
- Unsafe HTML/SVG embedding: inline SVGs can carry duplicate IDs or unsafe elements. Mitigate in the generator with `assert_safe_svg()` and `prefix_svg_ids()` before render; reject `base64`, `<image`, `<script`, and `<foreignObject>`.
- External asset dependency: previews could break from `file://` if they use `img src` paths. Mitigate by embedding committed SVG bodies inline while showing visible source-file links.
- Broken no-JS/browser rendering: generated output could require JavaScript before later page work. Mitigate by rendering semantic landmarks, nav, all sections, and inline previews as baseline HTML in this plan; later plans only progressively enhance.
- Premature milestone closeout: this plan does not update `.planning/PROJECT.md`, `.planning/STATE.md`, `.planning/ROADMAP.md`, or `.planning/REQUIREMENTS.md`.
- High severity block condition: any generator behavior that reads remote content, executes source-controlled SVG script, writes outside `brandbook/index.html`, or requires a non-stdlib build dependency blocks this plan.
</threat_model>
