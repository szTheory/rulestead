## Page Structure Contract

The first viewport must be the usable brand book, not a marketing landing page. It must show the Rulestead wordmark/mark, the tagline `Runtime decisions, made clear.`, the current theme state, and direct navigation into the source-driven sections.

Required sections, in order:

| Section | Required Content |
|---------|------------------|
| Overview | Brand essence, north-star message, audience frame, and brand mantra. |
| Voice and messaging | One-line description, pitches, messaging pillars, voice principles, copy examples, and UI-writing rules. |
| Color | Token-sourced light/dark palette, usage ratios, semantic states, decorative-only Signal Gold policy, and contrast notes. |
| Typography | Sora/Inter/IBM Plex Mono type ramp from the committed token/specimen system. |
| Logo | Light and dark wordmarks, mark variants, mono mark, favicon, and usage guardrails. |
| Layout and components | Grid, spacing, radius, panel, button, badge, code-block, and callout examples aligned to admin CSS tokens. |
| Iconography and imagery | Allowed themes, forbidden motifs, and committed specimen references. |
| Motion | Token timings, reduced-motion behavior, and permitted reveal patterns. |
| Assets and maintenance | Links or visible references to canonical source files, generator, drift-check, and size-budget policy. |

Layout:
- Desktop: constrained content width of `1200px` max, with a left or top section nav and a readable main column.
- Mobile: single-column content; section nav collapses to an in-page jump menu or horizontal anchor list.
- Use full-width bands or unframed sections; use cards only for repeated specimens, source-file references, or framed component examples.
- Do not nest UI cards inside cards.
- All SVG specimens must have stable dimensions with responsive constraints so loading, labels, and hover states cannot shift layout.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Theme control | Provide a segmented `System / Light / Dark` control matching the admin theme posture. Default to System, store only a brand-book-specific preference key if persistence is used, and reflect the active mode on the page wrapper. |
| Navigation | All major sections must have stable fragment IDs, visible focus states, and keyboard-reachable nav links. |
| Source links | Links to source artifacts should be plain anchors to repo-relative files where possible; no modal-only access to source references. |
| Asset previews | Logo/specimen previews may enlarge with a non-layout-shifting detail view, but the page must remain usable with JavaScript disabled. |
| Code/token copy | Optional copy buttons are allowed for token names and hex values; labels must be specific, such as `Copy token` or `Copy hex`. |
| Destructive actions | none; the generated page is read-only. |

Accessibility:
- Use semantic landmarks: header, nav, main, section, footer.
- Every non-decorative image or inline SVG needs an accessible name. Decorative separators and route-line ornament must be hidden from assistive technology.
- Focus rings must use the existing `--rs-focus-ring` pattern or equivalent token values.
- All normal-weight text must preserve the Phase 95 AA contrast decisions in both light and dark.
- Respect `prefers-reduced-motion`; animation and smooth scrolling must be disabled or reduced when requested.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, rule-line offsets, inline annotation gaps |
| sm | 8px | Compact element spacing, chip padding, tight token rows |
| md | 16px | Default element spacing, card padding, section internals |
| lg | 24px | Section padding, specimen group gaps |
| xl | 32px | Layout gaps, major content breaks |
| 2xl | 48px | Major section breaks, overview-to-detail transitions |
| 3xl | 64px | Page-level spacing on wide viewports |

Exceptions: existing token values `12px`, `20px`, `28px`, `36px`, and `40px` may be used only when sourced from `--rs-space-3`, `--rs-space-5`, `--rs-space-7`, `--rs-space-9`, or `--rs-space-10`; touch targets must be at least `44px`; layout max-width is `1200px`.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 15px (`--rs-text-base`) | 400 | 1.5 |
| Label | 12px (`--rs-text-xs` or mono annotation) | 600 | 1.35 |
| Heading | 22px (`--rs-text-xl`) | 600 | 1.2 |
| Display | 32px (`--rs-text-2xl` max) | 600 | 1.2 |

Rules:
- Use exactly these two weights for generated HTML text: 400 and 600. Existing outlined logo paths are exempt.
- Do not use monospace for body copy.
- Letter spacing must remain `0` for generated page text.
- Line lengths should stay between 60 and 78 characters for prose where layout permits.
- Do not introduce additional viewport-scaled type beyond existing token behavior.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Light `#f4f6f8`; Dark `#19222e` | Page background, broad content field, quiet section bands |
| Secondary (30%) | Light `#ffffff` / `#eef1f5`; Dark `#141c27` / `#1f2a38` | Cards, specimen frames, nav, code header surfaces |
| Accent (10%) | Light `#3A6F8F`; Dark `#5885a0`; warm accent `#9b5931` / `#ba6b3c` | Primary CTA, selected nav item, theme control active state, focus-adjacent callouts, active-route logo explanation, sparse highlight moments |
| Destructive | Light `#B44949` on white/rain tint, `#b04848` on Stone Mist; Dark `#bf6464` | Destructive/error copy only; no destructive page actions in this phase |

Accent reserved for: primary CTA, selected nav indicator, active theme segment, token callout emphasis, active route in the logo explanation, and at most one warm hero/detail highlight per viewport. Accent must not be applied to every link or every interactive element.

Additional color rules:
- Signal Gold `#D2A94E` is decorative-only; never use it as normal-weight text.
- Use Stead Blue as system confidence; use Ember Copper as moment-of-change emphasis.
- Avoid rainbow dashboards and generic neon developer-tool styling.
- Dark mode must keep the v1.13 base `#10161f` where the canonical dark-base token is required; do not swap it for Basalt `#0F1720`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Open brand assets |
| Empty state heading | Brand source unavailable |
| Empty state body | Regenerate the brand book from `brandbook/brand-book.md`, `brandbook/tokens.json`, and committed SVG assets before review. |
| Error state | Brand book generation failed. No source files were changed. Run the generator from the repo root and inspect the first missing or drifted path. |
| Destructive confirmation | none: the generated brand book is read-only |

Copy rules:
- Use clear, calm, exact, assured, practical language.
- Buttons must use verbs with consequences, such as `Open brand assets`, `Copy token`, `Copy hex`, and `View source`.
- Avoid vague controls such as `Continue`, `Submit`, `Go`, and `Done`.
- Error copy must state what happened, what did not happen, and what to do next.

## Motion Contract

| Motion | Contract |
|--------|----------|
| Durations | Use `150ms`, `200ms`, `320ms`, and `480ms` only, matching `--rs-motion-fast/base/slow/slower`. |
| Easing | Use `--rs-ease-standard`, `--rs-ease-out`, `--rs-ease-in`, or `--rs-ease-in-out`; reserve `--rs-ease-emphasis` for tiny confirmation feedback only. |
| Allowed patterns | Stepped reveal, layered fade/slide, path tracing, rule highlight sequencing, and diff transitions. |
| Forbidden patterns | Bouncing, looping glows, game-like jitter, exaggerated springs, and gimmicky 3D spins. |
| Reduced motion | Gate nonessential motion behind `prefers-reduced-motion: no-preference`; keep content visible and understandable without animation. |
