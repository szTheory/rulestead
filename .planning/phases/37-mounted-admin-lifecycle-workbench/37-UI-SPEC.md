## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4F5F2` | Page background, primary surfaces, table background, cleanup review canvas |
| Secondary (30%) | `#DCE3E6` | Filter rail blocks, cards, stat tiles, nav separators, row hover planes |
| Accent (10%) | `#2E6F95` | Active lifecycle preset, current environment pill, selected filter state, preview-entry link, post-action outcome banner link |
| Destructive | `#B24C34` | Archive confirmation, critical readiness, destructive warnings only |

Accent reserved for: active lifecycle preset chips, the current environment selector state, the primary "Review cleanup" CTA, preview route entry links, and the audit link inside the post-action outcome banner.

Additional semantic tones:
- Positive: `#4D7A5A` for `keep_active`, fresh evidence, and successful post-action return banners.
- Warning: `#C58A2E` for `needs_review`, partial evidence, unknown-heavy cleanup reviews, and production-policy caution.
- Muted text/border: `#68737A` for archived states, helper copy, and non-primary metadata.

Rules:
- Do not use accent blue for every link in the workbench. Ordinary secondary links stay neutral until hover.
- Reserve destructive ember for archive readiness badges, typed-confirm affordances, and the final archive action.
- Keep backgrounds light. This phase inherits the brand's calm operator posture, not a dark-mode-first treatment.

## Interaction Contract

### Canonical Workbench

- The existing `FlagLive.Index` inventory remains the only lifecycle workbench. Do not create a separate lifecycle LiveView.
- Add lifecycle entry presets as route-backed shortcuts that resolve into the same `/admin/flags` query schema.
- Keep all operator triage state URL-backed and canonicalized through `handle_params/3`.

### Canonical Query Contract

- Preserve existing params: `env`, `tenant`, `query`, `owner`, `lifecycle`, `stale`, `readiness`, `evidence_quality`, `include_archived`, `after`, `before`.
- Add `return_to` only for cross-route lifecycle actions. It must carry a full canonical path back to the originating workbench state.
- `owner` remains an exact `owner_ref` filter. Free-text discovery of `owner_display` or legacy labels stays inside `query`, not the owner filter.
- Filter copy must distinguish exact owner filtering from broader search.

### Screen Roles

- `FlagLive.Index`: scan, filter, and queue work. This screen owns lifecycle presets, owner triage, archived visibility, and entry into cleanup review.
- `FlagLive.Show`: calm read surface only. It may summarize lifecycle posture and expose a single "Review cleanup" CTA, but it does not host archive confirmation.
- `FlagLive.Cleanup`: canonical pre-mutation review surface. It owns evidence summary, blockers, unknowns, code references, recommendation posture, and the transition into explicit preview.
- New route-backed preview and confirm surfaces sit under cleanup, not under detail and not in a modal-only flow.

### Required Route Shape

- Use route-backed navigation for the lifecycle action spine:
  - `/admin/flags?...filters`
  - `/admin/flags/:key`
  - `/admin/flags/:key/cleanup`
  - `/admin/flags/:key/cleanup/preview`
  - `/admin/flags/:key/cleanup/confirm`
- Filter changes patch within the inventory LiveView.
- Transitions between inventory, detail, cleanup, preview, and confirm navigate between LiveViews.

### Archive Action Flow

- Primary operator path: `scan -> review cleanup -> preview archive -> confirm with reason -> return to queue with audit-linked outcome`.
- Preview must show:
  - lifecycle state
  - archive readiness
  - evidence quality
  - reasons, unknowns, blockers
  - code-reference summary
  - what changes after archive
  - clear statement that archival is explicit and not automatic
- Confirm must revalidate current lifecycle state before apply. If evidence or authored posture drifted, block apply and return the operator to preview with a clear revalidation message.
- No bulk lifecycle mutations. Every preview and confirm surface is scoped to one flag.

### Post-Action Return

- Successful archive returns to the exact canonical `return_to` workbench URL.
- If the originating queue excluded archived flags, inject or preserve `include_archived=true` for the return so the operator does not lose the flag.
- Show an outcome banner at the top of the returned workbench with:
  - archived flag key
  - environment
  - operator-entered reason echoed back
  - audit timeline link
- Highlight the archived row once on return. Do not keep the operator on the detail or cleanup page by default.

### Confirmation Rules

- Reason capture is required for every archive confirmation.
- Production confirmation requires exact typed key confirmation, matching the existing kill-switch precedent.
- Non-production confirmation requires a reason but not typed-key confirmation.
- Do not hide confirm behind hover-only affordances or transient toast-only success states.

## Component Inventory

| Surface | Contract |
|---------|----------|
| Lifecycle preset strip | Compact preset links above the existing filter grid; each preset writes canonical URL params only |
| Filter grid | Keep current dense form layout; add exact-owner helper copy and keep archived visibility as an explicit checkbox |
| Workbench table row | Preserve monospace key-first scanning, lifecycle/readiness/evidence badges, and add a clear cleanup-review entrypoint for archive candidates |
| Detail lifecycle cards | Keep existing card stack; add one primary "Review cleanup" CTA and no direct destructive button |
| Cleanup summary strip | Four-stat review summary: readiness, evidence quality, code references, evaluation evidence |
| Archive preview card stack | Reuse `rs-card`, `rs-stat`, and badge primitives; add before/after state summary and audit consequence text |
| Archive confirmation form | Reason textarea, typed-key field for production, disabled destructive submit until validation passes |
| Outcome banner | Neutral-success banner on return to workbench with audit link and one-time highlighted row |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Review cleanup |
| Empty state heading | No flags match this lifecycle view |
| Empty state body | Clear or widen the lifecycle, owner, readiness, or archived filters to restore the queue. Shared URLs preserve the current view exactly. |
| Error state | We couldn't load this lifecycle view. Refresh the page or reopen the shared URL. If cleanup evidence changed, rerun preview before archiving. |
| Destructive confirmation | Archive flag: Archive this flag only after reviewing code references, evaluation evidence, and blockers. Enter a reason to continue. |

Additional copy rules:
- Use `flag`, `archive`, `cleanup`, `owner`, `lifecycle state`, and `archive readiness` exactly as defined in the domain language guide.
- Do not say `delete`, `remove forever`, `zombie`, `stale bot`, or other novelty language.
- Use `Review cleanup` from both inventory and detail; do not split this into multiple verb variants.
- Outcome banner success copy: `Archived <flag_key> in <environment>. Review the audit timeline for the recorded reason.`

Destructive actions in scope:
- `Archive flag`:
  - confirmation approach: route-backed preview, route-backed confirm, reason required, typed key required in production, audit link shown after success
