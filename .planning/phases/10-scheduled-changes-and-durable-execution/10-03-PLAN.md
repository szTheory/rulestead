| Boundary | Description |
|----------|-------------|
| public scheduling facade -> governed action authorizer | Host policy and operator intent are enforced before durable scheduling here |
| scheduled execution -> underlying mutation helpers | Future-dated intent crosses into real publish/rollout/kill-switch mutation state here |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03-01 | E | direct scheduling path | mitigate | Restrict direct scheduling to the bounded action set and enforce `policy_bypass`/`emergency_bypass` checks through the existing authorizer seam, per D-04 and D-05. |
| T-10-03-02 | T | stale intent execution | mitigate | Validate archived resources, rollout stage targets, ruleset publishability, and kill-switch state before executing, and fail visibly with exact `failure_reason` values, per D-15 and D-16. |
| T-10-03-03 | R | actor provenance | mitigate | Preserve scheduling actor and explicit emergency reason on the scheduled record; runtime execution is recorded later as scheduler/system rather than impersonating the human, per D-12. |
| T-10-03-04 | D | scope expansion | mitigate | Reject any scheduling attempt outside `publish_ruleset`, `advance_rollout`, `engage_kill_switch`, and `release_kill_switch`; no ladders, recurrence, or arbitrary governed commands, per D-06 and D-07. |
