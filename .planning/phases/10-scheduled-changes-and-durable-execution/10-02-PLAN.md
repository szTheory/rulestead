<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| schedule write path -> Oban queue | Durable state and delivery intent cross into the background job system here |
| Oban worker -> governed mutation execution | A system actor executes a privileged change on behalf of prior human intent here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | T | worker payload | mitigate | Carry only `scheduled_execution_id` plus bounded metadata in job args and reload durable state before mutation, per D-03. |
| T-10-02-02 | R | retry/recovery path | mitigate | Append execution attempts and preserve the original `scheduled_execution_id` through retries and requeue, per D-10 and D-11. |
| T-10-02-03 | D | background retry loop | mitigate | Use bounded Oban retries and transition exhausted work to `quarantined` requiring explicit operator requeue, per D-08 and D-09. |
| T-10-02-04 | E | duplicate side effects | mitigate | Make execution idempotent by scheduled-execution identity and treat repeated completions as safe no-op success, per D-10. |
</threat_model>
