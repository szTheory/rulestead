Both are superseded by this phase's decisions. A downstream agent reading anchor doc without CONTEXT.md override will apply wrong config. | blocking | Always read `01-CONTEXT.md` BEFORE the anchor doc. CONTEXT.md flags supersessions in D-02 and D-06 + deferred D-20. |
| Workflow-level `paths-ignore` + required checks | GitHub documented behavior: workflow skipped by `paths-ignore` leaves required status checks in permanent Pending state, blocking PR merge forever | blocking | Use job-level `if:` conditionals with `dorny/paths-filter@v3` for anything feeding `release_gate`, NOT workflow-level `paths-ignore`. |
| Release-please + default GITHUB_TOKEN silent-CI-skip | GitHub documented: resources created by GITHUB_TOKEN don't trigger further Actions workflow runs. The v0.1.0 release PR opens but `ci.yml` never auto-runs against it. | advisory | Accept for Phase 1; document "manually re-run CI on release PR" in MAINTAINING.md.
