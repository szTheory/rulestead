## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| persisted snapshots -> ETS runtime cache | Serialized store state becomes hot-path evaluator input |
| runtime diagnostics/explain -> callers | Internal cache state is surfaced publicly and must stay bounded and redacted |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | T | ETS apply path | mitigate | Accept newer snapshot versions only and keep apply idempotent so bad or duplicate inputs do not replace known-good cache state. |
| T-04-05 | I | diagnostics/explain | mitigate | Limit runtime output to sanitized metadata such as version, age, and status; never surface raw attributes, targeting identifiers, or arbitrary ETS contents. |
| T-04-06 | D | runtime facade | mitigate | Keep lookup APIs explicit under `Rulestead.Runtime` so callers never accidentally trigger hidden stateful behavior through Phase 3 pure APIs. |
</threat_model>
