<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authoring publish path -> runtime snapshot persistence | Mutable authoring state becomes the immutable runtime artifact every evaluator node trusts |
| store adapters -> runtime refresh consumers | Adapter-private data must be normalized into one stable snapshot contract |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | T | snapshot publication | mitigate | Persist immutable versioned snapshots so failed or repeated publishes cannot silently rewrite the last-known-good payload. |
| T-04-02 | I | snapshot payload boundary | mitigate | Keep runtime snapshot fetches environment-scoped and bounded; do not leak authored rows or raw actor metadata into the persisted artifact. |
| T-04-03 | D | adapter parity | mitigate | Lock one shared contract suite so fake and Ecto adapters expose the same snapshot semantics before runtime caching depends on them. |
</threat_model>
