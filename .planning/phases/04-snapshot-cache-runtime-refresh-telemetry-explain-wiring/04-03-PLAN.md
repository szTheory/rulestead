<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime supervision -> host app startup | Library-owned runtime processes must not assume host-owned processes already started |
| refresh worker -> store and PubSub | Partial failures or missed notifications must degrade gracefully without corrupting cache state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | D | startup sequencing | mitigate | Supervise runtime ownership locally and allow empty/degraded boot so the evaluator never crashes because the control plane or PubSub is unavailable. |
| T-04-08 | T | refresh apply path | mitigate | Reject duplicate or older versions and preserve last-known-good state when fetch/compile/apply fails. |
| T-04-09 | D | PubSub dependency | mitigate | Treat PubSub as a wake-up path only and rely on polling reconciliation as the correctness backstop. |
</threat_model>
