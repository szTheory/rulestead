<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| runtime cache -> local filesystem backup | In-memory state is serialized to disk and must remain safe to restore |
| single-node runtime -> multi-node cluster | PubSub and polling coordination must converge without distributed shared-state assumptions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-10 | T | disk backup files | mitigate | Use versioned envelopes, checksum validation, safe decoding, and quarantine-on-corruption before any restore is accepted. |
| T-04-11 | D | offline restart | mitigate | Load disk backup opportunistically before refresh and keep backup failures isolated from in-memory last-known-good state. |
| T-04-12 | D | multi-node convergence | mitigate | Test cluster refresh explicitly with bounded latency assertions and version-based convergence, not shared mutable cache assumptions. |
</threat_model>
