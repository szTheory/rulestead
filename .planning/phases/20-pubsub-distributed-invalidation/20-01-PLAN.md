## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| core snapshot publisher -> notifier transport | Authoritative publish state crosses into a best-effort distributed event bus |
| host runtime config -> notifier adapter | Host-owned infrastructure names are injected into runtime wiring here |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-20-01-01 | T | invalidation payload | mitigate | Keep the notice minimal and versioned; receivers must re-fetch snapshots rather than trusting event contents as state. |
| T-20-01-02 | D | notifier publish path | mitigate | Make notifier use explicit config and degrade to no-op when PubSub is absent so publishing snapshots never fails because invalidation transport is unavailable. |
| T-20-01-03 | S | host PubSub wiring | mitigate | Require explicit `pubsub` config and avoid runtime autodetection of host infrastructure. |
| T-20-01-04 | I | event payload contents | mitigate | Never embed full snapshot payloads, changed rules, or actor data in invalidation notices. |
</threat_model>
