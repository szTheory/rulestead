<task id="57-03-01" type="execute">
  <name>Integrate threshold into Ecto apply pipeline</name>
  <read_first>
    - rulestead/lib/rulestead/store/ecto.ex (apply_audience_mutation Multi chain ~680-700)
    - rulestead/lib/rulestead/fake.ex (validate_blast_radius_threshold from Plan 57-02)
    - rulestead/lib/rulestead/governance/blast_radius_threshold.ex
  </read_first>
  <action>
In `rulestead/lib/rulestead/store/ecto.ex`:

1. Add alias: `alias Rulestead.Governance.BlastRadiusThreshold`

2. In the `apply_audience_mutation` Multi pipeline, in the `:preview` run step's `with` chain:
   - **Remove:** `:ok <- ensure_protected_audience_confirmation(command)`
   - **Add after** `:ok <- ensure_fresh_audience_preview(command, preview)`:
     ```elixir
     :ok <- validate_blast_radius_threshold(command, preview)
     ```

3. Implement `validate_blast_radius_threshold/2` matching Fake:
   ```elixir
   defp validate_blast_radius_threshold(command, preview) do
     dependency_entries = audience_dependency_entries(preview, command)

     BlastRadiusThreshold.validate_protected_apply(
       command,
       preview,
       dependency_entries: dependency_entries
     )
   end
   ```

4. **Remove** `ensure_protected_audience_confirmation/1` function heads from ecto.ex

5. On threshold block in dependency_validation or dedicated step: ensure audit event records blast radius metadata (match Fake fields: `blast_radius_verdict`, `blast_radius_breach_reasons`, `reference_count`)

6. Verify `audience_dependency_entries/2` exists in ecto.ex (or equivalent) — reuse same normalization as Fake.
  </action>

  </verify>

    - `mix compile --warnings-as-errors` exits 0
  </acceptance_criteria>
</task>
