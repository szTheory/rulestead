# Plan 57-01: Pure BlastRadiusThreshold Evaluator

<objective>
Implement `Rulestead.Governance.BlastRadiusThreshold` as a pure module with `assess/2` and `validate_protected_apply/3`, covering all verdict paths (below, above, indeterminate) per Phase 57 CONTEXT D-01 through D-05 and D-08.
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-57-01: False below-threshold on missing data | high | Fail-closed `:indeterminate` for all missing/stale/unresolved inputs (D-04) |
| T-57-02: Threshold bypass via non-prod misclassification | medium | Use `Compare.protected_target?/1`; unknown env → protected |
| T-57-03: Population count leakage in assessment | low | Set `authoritative_population_count?: false` always; no impression fields |
</threat_model>

  <action>
Create `rulestead/lib/rulestead/governance/blast_radius_threshold.ex` with:

**Module:** `Rulestead.Governance.BlastRadiusThreshold`

**Public API:**
- `@spec assess(map(), keyword()) :: {:ok, assessment()} | {:error, Rulestead.Error.t()}`
- `@spec validate_protected_apply(map(), map(), keyword()) :: :ok | {:error, Rulestead.Error.t()}`

**Opts (keyword, test-only overrides per D-08):**
- `:threshold_profile` — default `:default`
- `:protected_environment?` — override; when omitted, derive from `Compare.protected_target?(environment_key)`

**Default profile `@default_profile` thresholds (protected env only):**
- `:indeterminate` if any D-04 trigger fires
- `:above_threshold` if `operation == "archive"` and `reference_count > 0`
- `:above_threshold` if `operation == "update"` and `reference_count > 2`
- otherwise `:below_threshold`

**Inputs to assess/2** (map keys):
- `environment_key`, `operation`, `preview_fingerprint`, `preview_schema_version`
- `affected_references` (list from preview)
- `affected_reference_keys` (from command, optional for mismatch check)
- `dependency_entries` (optional, for DependencyValidator)
- `hidden_reference_count` (optional, default 0)

**Assessment map fields:**
```elixir
%{
  verdict: :below_threshold | :above_threshold | :indeterminate,
  reference_count: integer(),
  distinct_flag_count: integer(),
  rollout_hints: [map()],
  lifecycle_hints: [map()],
  threshold_profile: :default,
  operation: String.t(),
  environment_key: String.t(),
  preview_fingerprint: String.t(),
  breach_reasons: [%{code: String.t(), observed: term(), limit: term(), remediation: String.t()}],
  protected_environment?: boolean(),
  authoritative_population_count?: false
}
```

**Finding codes (use ImpactPreview.finding/4 pattern):**
- `blast_radius_above_threshold`
- `blast_radius_indeterminate`
- `blast_radius_missing_preview_inputs`
- `blast_radius_unresolved_dependency_truth`

**validate_protected_apply/3:**
- Args: `command` (ApplyAudienceMutation map), `preview` (ImpactPreview build result), `opts`
- When NOT protected env: return `:ok` regardless of verdict (D-02 non-protected bypass)
- When protected AND verdict in `[:above_threshold, :indeterminate]`: return `{:error, StoreError.invalid_command(message, metadata: assessment_metadata, details: findings)}`
- Above-threshold remediation string: "This audience change affects more than the direct-apply limit for protected environments. Submit a change request for governed approval."
- Indeterminate remediation: "Blast radius cannot be evaluated safely. Re-run preview after resolving dependency visibility, or cancel the change."

**Indeterminate triggers (all must block in protected env):**
- blank `preview_fingerprint`
- `preview_schema_version != ImpactPreview.schema_version()`
- `affected_reference_keys` mismatch vs `AudienceDependencies.reference_keys(affected_references)` when references expected
- `DependencyValidator.blockers?(findings)` when dependency_entries provided
- any reference with `rollout_context.available? == false` or `lifecycle_context.available? == false`
- `hidden_reference_count > 0`

Use `@moduledoc` documenting default thresholds and opts override posture (no Rulestead.Config in Phase 57).
