## Scope Guardrails

| Guardrail | Contract |
|-----------|----------|
| Phase role | Treat Phase 118 as milestone evidence and idempotence closeout, not as a new visual polish, route IA, foundation, component, or visual-regression infrastructure phase. |

| Planning truth | Mark VER-01 through VER-04 complete only after evidence exists. Update planning files during execution/verification, not speculatively during planning. |

## Spacing Scale

Declared values for Phase 118 evidence docs, review tables, test-facing labels, and any narrow evidence surface adjustments (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px / `--rs-space-1` | Icon gaps, inline metadata gaps, artifact-status glyph spacing |
| sm | 8px / `--rs-space-2` | Compact row spacing, command lists, evidence table cell padding |
| md | 16px / `--rs-space-4` | Default evidence section spacing, form/control group spacing in tested surfaces |
| lg | 24px / `--rs-space-6` | Evidence map section padding, mobile stack gaps, screenshot review grouping |
| xl | 32px / `--rs-space-8` | Major evidence cluster separation between matrix, workflow, guard, and planning proof |
| 2xl | 48px | Large closeout document breaks and milestone-level proof grouping |
| 3xl | 64px | Page-level separation only for generated/admin evidence surfaces that already use this rhythm |

Exceptions: Existing shipped component internals keep their current tokenized spacing. `--rs-touch-target-min` / 44px remains the exception for icon-only controls, shell controls, command-palette controls, destructive-flow controls, and compact mobile actions. Playwright artifact file paths and terminal commands may wrap or scroll inside their local code/table container rather than forcing extra page spacing.

## Typography

Declare these four roles for evidence maps, verification docs, generated review surfaces, and any test-facing UI text touched by Phase 118. Existing component internals keep their shipped typography unless a deterministic evidence failure requires a narrow fix using these roles.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | `--rs-text-base` / 0.95rem | `--rs-weight-normal` / 400 | `--rs-leading-normal` / 1.5 |
| Label | `--rs-text-sm` / 0.86rem | `--rs-weight-semibold` / 600 | `--rs-leading-snug` / 1.35 |
| Heading | `--rs-text-lg` / 1.15rem | `--rs-weight-semibold` / 600 | `--rs-leading-tight` / 1.2 |
| Display | `--rs-text-2xl` / clamp(1.5rem, 2vw, 2rem) | `--rs-weight-semibold` / 600 | `--rs-leading-tight` / 1.2 |

Use only these two weights for new or revised Phase 118 surfaces: 400 and 600. Monospace values for file paths, command names, route paths, screenshot artifact patterns, JSON, and source markers use `--rs-font-mono` within Body or Label roles above.

## Color

Use the existing theme-scoped admin tokens. Phase 118 must preserve light, explicit dark, and system-dark behavior without introducing new palette values.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--rs-bg`, `--rs-surface` | Shell background, evidence pages, matrix and workflow surfaces under test, main verification panels |
| Secondary (30%) | `--rs-surface-muted`, `--rs-surface-faint`, `--rs-border`, `--rs-border-subtle`, `--rs-text-muted` | Evidence tables, secondary proof context, command notes, artifact metadata, guard output summaries |
| Accent (10%) | `--rs-primary` | Current evidence focus, active route/subnav indicator, primary proof command, selected command-palette option, visible focus emphasis |
| Destructive | `--rs-error`, `--rs-critical`, `--rs-error-soft`, `--rs-error-border` | Kill-switch evidence, destructive confirmation proof, blocked destructive state, and risky-action verification only |

Accent reserved for: the primary evidence action, active matrix or workflow sample indicator, focused shell/route controls, selected command-palette option, current verification step, and existing route-active affordances. It is not for every link, every button, decorative highlights, or generic status coloring.

The existing Ember Copper `--rs-accent` remains a restrained brand and risky-state marker where components already use it: lockup route node, timeline/highlight details, risky-state emphasis, and authored-state markers. Do not broaden it into a second general interaction accent.

## Evidence Interaction Contract

| Area | Contract |
|------|----------|
| Evidence map | Produce a compact Phase 118 evidence map with rows for requirement, surface, assertion type, command, artifact pattern, status, intentional exception, and residual risk. |
| Matrix screenshots | Preserve established artifact naming: `ui-matrix-${section}-${theme}-${viewport}-${motion}.png`. Cover light, dark, system-dark, desktop, mobile, and targeted reduced-motion cases where they affect the surface. |
| Workflow screenshots | Preserve established artifact naming: `flow-${route}-${theme}-${viewport}.png`. Cover overview, inventory, rules, kill, audience, audit, explain, and simulate unless a source-backed equivalent set is recorded. |
| Reduced motion | Keep reduced-motion evidence targeted and behavioral. Prove nonessential transform/animation is absent; do not multiply every route by every motion mode. |
| Keyboard and focus | Browser proof must cover representative rail/home navigation, command palette open/search/result selection, filters/forms, contextual subnav/tabs, and at least one destructive or review workflow. |
| ARIA and roles | Prefer Playwright role locators and accessible names for regions, headings, forms, navigation, command-palette options, tables, and destructive controls. Avoid selector-only proof when a semantic role is expected. |
| Overflow | Keep the no-page-overflow contract at `document.documentElement.scrollWidth - clientWidth <= 1`; long keys, artifact paths, raw details, and commands must wrap or local-scroll inside their boundary. |
| Fixture health | Use ExUnit/source assertions for deterministic fixture availability, real component usage, route examples, rare states, and dev/test matrix isolation. |
| Contrast | Keep selected contrast checks in static fixture and guard-script layers. Do not introduce exhaustive runtime route-pixel contrast auditing. |
| Review ergonomics | Evidence docs should be dense and scannable: status first, proof command next, artifact pattern next, exception last. Do not use marketing-style hero layouts or explanatory cards for closeout docs. |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Capture Evidence |
| Empty state heading | No evidence captured yet |
| Empty state body | Run the focused Phase 118 proof command, then record the generated screenshot pattern and deterministic assertion result. |
| Error state | Evidence could not be collected. Check the demo backend URL, fixture health, route path, and guard output, then rerun the focused proof. |
| Destructive confirmation | Kill switch evidence: Confirm the affected flag key, provide a reason, verify the disabled and unavailable states, and keep `Return to evidence map` as the non-danger path. |

Additional copy rules:

- Use proof-specific verbs: `Capture Evidence`, `Record Guard Output`, `Review Artifact`, `Verify Route`, `Verify Matrix`, `Record Exception`, `Close Requirement`.
- Avoid generic labels such as `Submit`, `Go`, `OK`, `Done`, or `Run` when a more specific evidence action is available.
- Empty, unavailable, denied, blocked, and error copy must name the missing proof, likely source of failure, and next safe command or artifact to inspect.
- Exception copy must identify the requirement, surface, reason, reviewer expectation, and whether the exception is temporary or intentionally durable.
- Planning closeout copy must not claim VER-01 through VER-04 completion until the matching command and artifact evidence exist.
