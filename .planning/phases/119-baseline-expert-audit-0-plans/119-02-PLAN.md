## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub run metadata -> audit metrics | Recent run/job timing is live external metadata and may change while the audit is written. |
| Local Mix diagnostics -> audit recommendations | Local machine results may differ from CI runner behavior; the audit must label local versus CI evidence. |
| Release workflow source -> security posture | Publish workflows expose trust-critical behavior and secret boundaries that must not be weakened by speed recommendations. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-119-05 | Repudiation | CI timing sample | mitigate | Record run IDs, event, branch, conclusion, and command used for each live sample; label p95 as unavailable when sample size is insufficient. |
| T-119-06 | Tampering | Cache and PLT posture | mitigate | Inventory cache keys and restore breadth against OS, Elixir, OTP, lockfile, `.tool-versions`, `MIX_ENV`, and package scope; do not edit cache keys in Phase 119. |
| T-119-07 | Information disclosure | `publish-hex` and `HEX_API_KEY` | mitigate | Mention secret names and protected environment boundaries only; never print secret values. |
| T-119-08 | Denial of service | Diagnostic commands | accept | Commands are local baseline diagnostics required by D-11; if environment prerequisites fail, record failure text instead of retrying indefinitely or changing tests. |
| T-119-SC | Tampering | npm/pip/cargo installs | accept | This plan performs no package-manager install tasks and adds no dependencies. |
</threat_model>
