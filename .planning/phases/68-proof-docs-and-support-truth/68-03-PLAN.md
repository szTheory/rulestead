In `prompts/rulestead-host-app-integration-seam.md`, add section after guarded-rollout auto-advance (or after Oban/workers if auto-advance section absent):

**Heading:** `## Host preview evidence (audience impact previews)`

**Content must cover:**
1. **Opt-in resolver:** Host implements `Rulestead.PreviewEvidenceResolver` (or configured module) via `config :rulestead, :preview_evidence_resolver, MyApp.PreviewEvidence`
2. **No resolver configured:** Pre-v1.9 behavior — previews use authored state and explicit samples only
3. **Bounded summaries:** `sample_evidence` rows (allowlisted fields: `actor_key`, `targeting_key`, `matched?`, `reason`) and `impression_evidence` (`window_label`, `sampled_impressions`, `matched_impressions`, optional `variant_breakdown`)
4. **Redaction:** Resolver must not return raw PII traits; core redacts before fingerprint
5. **Fail-closed:** Invalid, oversized, or policy-denied resolver results return `Rulestead.Error` — admin shows alert, no silent fallback to authoritative counts
6. **Non-claims:** Rulestead does not ingest impressions, run warehouse queries, or provide fleet/population analytics dashboards

Include config example:
```elixir
config :rulestead, :preview_evidence_resolver, MyApp.RulesteadPreviewEvidence
```
