## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| compose environment -> Phoenix runtime config | `DEMO_FRONTEND_URL` crosses from local/demo deployment configuration into backend-rendered HTML. |
| Phoenix rendered HTML -> browser navigation | Users click generated absolute links to the sibling FleetDesk frontend origin. |
| test application env -> rendered HTML | Tests override app config and must restore it to avoid cross-test contamination. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-112.1-01 | Spoofing | `RulesteadDemo.DemoUrls.fleetdesk_frontend_url/0` | mitigate | Read only runtime application config sourced from `DEMO_FRONTEND_URL`; do not read request params, headers, origin, or CORS config. Task 1 negative source assertions enforce this. |
| T-112.1-02 | Tampering | `home.html.heex`, `layouts.ex` | mitigate | Use HEEx attribute rendering with `@fleetdesk_frontend_url`; backend tests prove non-3000 configured values render and hardcoded port strings are removed. |
| T-112.1-03 | Repudiation | backend regression evidence | mitigate | Task 3 records automated ExUnit proof for configured URL rendering and copy removal. |
| T-112.1-04 | Information Disclosure | rendered absolute FleetDesk URL | accept | The selected demo frontend origin is already printed/exported by demo scripts and intentionally exposed as a navigation target; no secrets are introduced. |
| T-112.1-05 | Denial of Service | app env override in tests | mitigate | Keep env-mutating tests non-async and restore/delete previous config in `on_exit/1`. |
| T-112.1-06 | Elevation of Privilege | admin route/session handling | accept | The plan does not change auth/session/admin authorization routes or policies. |
| T-112.1-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install task exists in this plan; existing dependencies are reused. |

High-severity threats: none identified. The URL is deployment/demo configuration, not user request input, and the plan does not alter authentication, authorization, persistence, public APIs, schemas, or release workflows.
