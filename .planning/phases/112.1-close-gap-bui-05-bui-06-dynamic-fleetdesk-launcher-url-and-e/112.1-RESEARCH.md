```text
  -> PageController renders :home with fleetdesk_frontend_url assign
```

### Pattern 1: Runtime Config Helper

**What:** Add a tiny demo helper, for example `RulesteadDemo.DemoUrls.fleetdesk_frontend_url/0`, that reads `Application.get_env(:rulestead_demo, :fleetdesk_frontend_url, "http://localhost:3000")`. [CITED: https://elixir.hexdocs.pm/Application.html]

**When to use:** Use when controller/layout rendering needs the external sibling frontend URL. [VERIFIED: `home.html.heex`; VERIFIED: `layouts.ex`]

**Example:**

```elixir
# Source: https://elixir.hexdocs.pm/Application.html
defmodule RulesteadDemo.DemoUrls do
  @fallback_fleetdesk_frontend_url "http://localhost:3000"

  def fleetdesk_frontend_url do
    Application.get_env(
      :rulestead_demo,
      :fleetdesk_frontend_url,
      @fallback_fleetdesk_frontend_url
    )
  end
end
```

Do not store this value in a module attribute by calling `Application.get_env/3` at module load time; runtime config should be read inside a function. [CITED: https://elixir.hexdocs.pm/Application.html]

### Pattern 2: Controller Assigns into HEEx

**What:** Render `:home` with `fleetdesk_frontend_url: DemoUrls.fleetdesk_frontend_url()`. [CITED: https://phoenix.hexdocs.pm/Phoenix.Controller.html]

**When to use:** Use for the launcher card because Phoenix controllers render templates with assigns. [CITED: https://phoenix.hexdocs.pm/Phoenix.Controller.html; VERIFIED: `PageController.home/2`]

**Example:**

```elixir
# Source: https://phoenix.hexdocs.pm/Phoenix.Controller.html
def home(conn, _params) do
```

### Pattern 4: Normalized Playwright URL Assertion

**What:** Normalize expected `DEMO_FRONTEND_URL` with JavaScript `new URL(...)` and assert both Phoenix-rendered links have matching `href` values. [CITED: https://playwright.dev/docs/api/class-locatorassertions]

**When to use:** Use in `brand-ui-evidence.spec.ts` after `page.goto(backendUrl)`. [VERIFIED: `brand-ui-evidence.spec.ts`; VERIFIED: `playwright.config.ts`]

**Example:**

```typescript

await expect(page.getByRole("link", { name: "FleetDesk app" })).toHaveAttribute(
  "href",
```

### Anti-Patterns to Avoid

- **Deriving FleetDesk URL from Phoenix endpoint or request origin:** FleetDesk is a sibling frontend, not the Phoenix endpoint. [VERIFIED: `112.1-CONTEXT.md`; VERIFIED: `docker-compose.proxy.yml`]
- **Using CORS config as the rendered URL source:** CORS is an allowlist and may contain multiple origins; it is not the canonical navigation target. [VERIFIED: `runtime.exs`; VERIFIED: `RulesteadDemoWeb.Plugs.DemoCors`; ASSUMED]
- **Adding a global plug for two links:** It widens request state and hides the narrow demo dependency. [VERIFIED: `112.1-CONTEXT.md`]
- **Adding broad visual baselines:** The milestone explicitly prefers curated browser evidence, not every-route pixel baselines. [VERIFIED: `.planning/REQUIREMENTS.md`; VERIFIED: `112.1-CONTEXT.md`]
- **Editing FleetDesk branding or admin polish:** Out of scope for this inserted phase. [VERIFIED: `112.1-CONTEXT.md`; VERIFIED: `.planning/REQUIREMENTS.md`]

## Common Pitfalls

### Pitfall 1: Reading Runtime Config at Compile Time

**What goes wrong:** A module attribute captures the fallback URL during compilation and ignores later `DEMO_FRONTEND_URL` runtime values. [CITED: https://elixir.hexdocs.pm/Application.html]

**Why it happens:** `Application.compile_env/3` and module attributes are compile-time paths; this phase needs request-time/runtime application config. [CITED: https://elixir.hexdocs.pm/Application.html]

**How to avoid:** Read with `Application.get_env/3` inside a function called while rendering. [CITED: https://elixir.hexdocs.pm/Application.html]

**Warning signs:** Tests pass only with `localhost:3000`, or changing `Application.put_env/3` in a test does not affect rendered HTML. [ASSUMED]

### Pitfall 2: Only Fixing the Launcher Card

**What goes wrong:** The landing card works but the shared navbar still sends users to `http://localhost:3000`. [VERIFIED: `home.html.heex`; VERIFIED: `layouts.ex`]

**Why it happens:** The same hardcoded URL appears in both `home.html.heex` and `layouts.ex`. [VERIFIED: codebase grep]

**How to avoid:** Add backend and Playwright assertions for both rendered links. [VERIFIED: `112.1-CONTEXT.md`]

**Warning signs:** `rg "localhost:3000" examples/demo/backend/lib/rulestead_demo_web` still finds rendered links after implementation. [VERIFIED: codebase grep]

### Pitfall 3: Comparing Raw URL Strings Without Normalization

**What goes wrong:** Browser-resolved `href` includes a trailing slash or normalized host form while env input does not. [ASSUMED]

**Why it happens:** DOM `HTMLAnchorElement.href` resolves/normalizes URLs. [ASSUMED]

**How to avoid:** Normalize expected values with `new URL(value).href` before comparing in Playwright. [ASSUMED; CITED: https://playwright.dev/docs/api/class-locatorassertions]

**Warning signs:** CI failure shows expected `http://127.0.0.1:12345` and actual `http://127.0.0.1:12345/`. [ASSUMED]

### Pitfall 4: Evidence Proves Independent Pages, Not Navigation

**What goes wrong:** Playwright renders Phoenix and FleetDesk separately but never proves the launcher reaches the selected FleetDesk origin. [VERIFIED: current `brand-ui-evidence.spec.ts`]

**Why it happens:** Existing evidence spec has separate launcher and FleetDesk tests. [VERIFIED: current `brand-ui-evidence.spec.ts`]

**How to avoid:** Add one click-through test from the Phoenix launcher and assert FleetDesk host-brand content after navigation. [VERIFIED: `112.1-CONTEXT.md`; CITED: https://playwright.dev/docs/test-assertions]

**Warning signs:** The test can pass if the launcher still points at `localhost:3000` while FleetDesk independently renders at `DEMO_FRONTEND_URL`. [VERIFIED: `.planning/v1.16-MILESTONE-AUDIT.md`]

### Pitfall 5: Widening Scope Into Brand/Admin Polish

**What goes wrong:** The inserted phase becomes a UI polish phase and risks invalidating v1.16 locked brand boundaries. [VERIFIED: `112.1-CONTEXT.md`; VERIFIED: `.planning/REQUIREMENTS.md`]

**Why it happens:** The audit also mentions build/release evidence warning; it is easy to overcorrect beyond the narrow blocker. [VERIFIED: `.planning/v1.16-MILESTONE-AUDIT.md`]

**How to avoid:** Only add the one curated build/release route if the plan includes D-10; do not restyle or refactor admin surfaces. [VERIFIED: `112.1-CONTEXT.md`]

**Warning signs:** Changes touch `rulestead_admin` CSS/components, brand tokens, release workflows, schemas, or public APIs. [VERIFIED: `112.1-CONTEXT.md`; VERIFIED: AGENTS.md]

### Controller Regression Test Shape

```elixir
# Source: existing ConnCase pattern in page_controller_test.exs
test "GET / renders configured FleetDesk URL", %{conn: conn} do
  previous = Application.get_env(:rulestead_demo, :fleetdesk_frontend_url)
  Application.put_env(:rulestead_demo, :fleetdesk_frontend_url, "http://127.0.0.1:3999")

  on_exit(fn ->
    if previous do
      Application.put_env(:rulestead_demo, :fleetdesk_frontend_url, previous)
    else
      Application.delete_env(:rulestead_demo, :fleetdesk_frontend_url)
    end
  end)

  conn = get(conn, ~p"/")
  html = html_response(conn, 200)
  assert html =~ ~s(href="http://127.0.0.1:3999")
  refute html =~ ~s(href="http://localhost:3000")
end
```
