# Phase 112.1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Existing Phoenix HEEx + Tailwind CSS 4 + vendored daisyUI plugins for the Phoenix launcher; existing Next.js CSS modules/global CSS for FleetDesk |
| Icon library | Existing vendored Heroicons classes in the Phoenix demo; no new icons required |
| Font | Existing system UI stack for FleetDesk; existing launcher/admin font stack remains unchanged |

No shadcn initialization for this phase. Phase 107 and v1.16 requirements explicitly prohibit new component libraries or CSS build systems, and Phase 112.1 is limited to dynamic URL plumbing, copy correction, and browser evidence.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, compact evidence-only labels |
| sm | 8px | Compact link/nav gaps and small text spacing |
| md | 16px | Default launcher card gaps, grid gaps, and responsive page padding |
| lg | 24px | Card padding and major content grouping |
| xl | 32px | Header-to-card spacing and launcher section rhythm |
| 2xl | 48px | Page-level vertical padding on the launcher |
| 3xl | 64px | Reserved for existing page-level spacing only; do not add new large spacing in Phase 112.1 |

Exceptions: Keep existing Phoenix/daisyUI radius and control sizing tokens. Maintain the current theme-toggle and nav button hit areas; do not restyle FleetDesk or admin spacing while fixing URLs.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.5 |
| Heading | 18px | 600 | 1.2 |
| Display | 30px | 600 | 1.2 |

Use only regular 400 and semibold 600 for any touched copy. Existing heavier weights may remain where already present, but Phase 112.1 must not introduce new typography styles or widen the type scale.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Light `#f5f7f6`; dark `#10161f` | Phoenix demo launcher page background and Rulestead-owned chrome surfaces |
| Secondary (30%) | Light `#ffffff`; dark `#141c27` | Launcher cards, navbar surfaces, and evidence screenshots for Rulestead-owned demo chrome |
| Accent (10%) | Stead Blue `#3A6F8F`; Ember Copper `#9b5931` | Link hover/focus affordances, primary guide link, and existing Rulestead launcher accents only |
| Destructive | `#B44949` | Destructive actions only; none are introduced in this phase |

Accent reserved for: FleetDesk link focus/hover state in Phoenix-owned launcher chrome, Adoption guide primary action, text links inside the launcher API card, and existing focus rings. Do not apply Rulestead accent colors inside the FleetDesk Next.js host app.

FleetDesk remains host-branded and keeps its existing `--fd-*` token system: dominant `#f6f8fb` / `#121820`, surface `#ffffff` / `#18212b`, brand `#2f4f6f` / `#86a2bd`, and accent `#6f7f4f` / `#b5c080`. No FleetDesk palette, logo, or layout changes are part of this phase.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Launcher card link | The Phoenix launcher card for FleetDesk must navigate in the same tab to the configured `DEMO_FRONTEND_URL`, normalized as a browser `href`. |
| Layout nav link | The shared Phoenix layout `FleetDesk` nav link must use the same configured URL as the launcher card. |
| URL source | `DEMO_FRONTEND_URL` is the canonical external FleetDesk URL. `http://localhost:3000` is only the unaided local fallback exposed through Phoenix runtime/application config. |
| Copy correction | Remove copy that says FleetDesk is on "port 3000". Use URL-neutral language that still explains FleetDesk is the sample customer dispatch product. |
| Evidence behavior | Playwright must assert both rendered `href` values match `DEMO_FRONTEND_URL`, then click through from the Phoenix launcher to the live FleetDesk origin and verify FleetDesk host-brand content. |
| Navigation target | Keep links same-tab. This is normal demo navigation, not an outbound marketing link. |
| Responsive behavior | Existing desktop/mobile launcher and FleetDesk evidence must continue to show no horizontal overflow. |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Open FleetDesk app |
| Empty state heading | Not applicable |
| Empty state body | The launcher always renders its three cards; no empty state should be added for Phase 112.1. |
| Error state | FleetDesk link is not configured. Set `DEMO_FRONTEND_URL` or use the local fallback. |
| Destructive confirmation | None. Phase 112.1 introduces no destructive actions. |

Required visible copy update for the FleetDesk launcher card body:

`Sample customer dispatch product. Flags change what dispatchers see.`

Do not introduce marketing claims, Rulestead branding inside FleetDesk, or new explanatory UI text beyond the URL-neutral copy correction.

## Evidence Contract

| Evidence | Required Assertion |
|----------|--------------------|
| Phoenix launcher screenshot | Rulestead logo is visible; FleetDesk host heading remains visible; no horizontal overflow at desktop and mobile widths. |
| FleetDesk link hrefs | Launcher card link and layout nav link both equal `new URL(DEMO_FRONTEND_URL).href`. |
| Click-through proof | Clicking the Phoenix launcher FleetDesk card lands on the selected FleetDesk origin and shows `.fd-brand-name` = `FleetDesk` and `.fd-brand-mark` = `FD`. |
| FleetDesk host screenshots | FleetDesk remains host-branded in light and system-dark evidence; no Rulestead wordmark appears inside FleetDesk. |
| Build/release warning coverage | If included in the implementation plan, add `/admin/flags/fleet-map-v2/rollouts?env=staging` to the existing brand evidence matrix without restyling admin screens. |

Screenshots remain supporting artifacts. Do not add broad pixel baselines or visual-regression snapshots for every route.
