## Integration And Flow Check

### Confirmed Wiring

- Runtime tenant propagation stays on the shared core seam through `Rulestead.Tenancy`, `Rulestead.Context`, Plug, LiveView, Oban, and evaluator bucketing.
- Reviewed-artifact import/promotion/apply wiring persists `tenant_key` and revalidates reviewed scope before mutation.
- Compare and persistence paths accept `tenant_key` in both Ecto and fake adapters, and immutable environment versions retain tenant provenance.
- Phase 28 demo seams remain intact: `/api/flags`, `/api/flags/stream`, and `/demo/sign-in` are unchanged, and the external frontend still consumes those routes.

### Critical Gaps

- `TEN-01`, `TEN-03`: [`rulestead_admin/lib/rulestead_admin/live/session.ex`](/Users/jon/projects/rulestead/rulestead_admin/lib/rulestead_admin/live/session.ex:40) resolves actor and environment only, while compare pages call `Rulestead.compare_environments/3` without `tenant_key` in [`environment_compare_live/index.ex`](/Users/jon/projects/rulestead/rulestead_admin/lib/rulestead_admin/live/environment_compare_live/index.ex:148) and [`show.ex`](/Users/jon/projects/rulestead/rulestead_admin/lib/rulestead_admin/live/environment_compare_live/show.ex:140). The mounted-admin tenant seam is therefore incomplete in operator flows.
- `TEN-03`: [`rulestead/lib/rulestead/store/ecto.ex`](/Users/jon/projects/rulestead/rulestead/lib/rulestead/store/ecto.ex:2658) and [`rulestead/lib/rulestead/fake.ex`](/Users/jon/projects/rulestead/rulestead/lib/rulestead/fake.ex:1887) build audit metadata from `command.metadata` only, while [`rulestead/lib/rulestead/audit_event.ex`](/Users/jon/projects/rulestead/rulestead/lib/rulestead/audit_event.ex:33) serializes whatever context it is given. Audit records can therefore miss `tenant_key` unless callers inject it manually.
