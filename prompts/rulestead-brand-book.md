I’ve put the full Rulestead Brand Book into the canvas.

The naming screen still supports the choice: on a quick public web pass, Rulestead mostly surfaced surname/person pages rather than an obvious active software brand, which is a much cleaner starting point than category-native names. In the Elixir lane, FunWithFlags is still the established Hex package, and the broader feature-management space is already crowded with brands like LaunchDarkly, Unleash, Flagsmith, Flipt, FeatureHub, ConfigCat, DevCycle, Split, and others, so the book deliberately pushes Rulestead away from literal “flag/config/launch” tropes.  ￼

What’s in the brand book:

* full brand strategy and positioning
* naming rationale for Rulestead
* messaging architecture and tagline direction
* brand voice, tone, and copy rules
* color system with hex palette
* typography stack
* logo direction and visual metaphor territory
* imagery, motion, iconography, and layout guidance
* UI writing standards and documentation style
* OSS/community posture and implementation defaults

The core stance I locked in is:
calm, infrastructure-grade, Elixir-native, explainable, steady under operational pressure.

The headline recommendations are:

* Brand character: structured, lucid, trustworthy, never hypey
* Tagline default: Runtime decisions, made clear.
* Color posture: mineral/dark neutrals, a controlled blue, and a restrained ember accent
* Type system: Sora + Inter + IBM Plex Mono
* Visual metaphor: paths, layers, topology, ordered systems — not literal flags or phoenix mascots

One important caveat: this is a brand strategy and public-web distinctness screen, not formal trademark clearance or legal opinion.
