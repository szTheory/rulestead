# Security Policy

## Reporting a vulnerability

Please do not open a public GitHub issue for a suspected security issue.

Instead, report it privately to `security@rulestead.dev` with:

- a description of the issue
- impact assessment
- reproduction steps or proof of concept
- any relevant logs or screenshots with secrets removed

The maintainer will acknowledge receipt, reproduce the issue privately,
and coordinate a fix and disclosure timeline before any public write-up.

## Scope and support policy

Rulestead is pre-1.0 and not yet published. During this phase:

- there are no long-term support branches
- fixes land on the active development line only
- there is no backport policy

Once external releases begin, the supported-version policy will be updated
here.

## Security posture

The project is being built with an explicit threat model and audit trail
expectations. See the planning artifacts and later security-focused prompts
for the evolving posture while the implementation is still in flight,
including `.planning/phases/01-repo-bootstrap/01-04-PLAN.md`.
